# 19.11 - SQS FIFO Queues

## Key Concepts

### 1. FIFO = First In, First Out (Ordering Guarantee)
- Messages are consumed in the **exact order** they were sent
- Standard queues: best-effort ordering (can be out of order)
- FIFO queues: **strict ordering guarantee**

```
Producer sends:   1 → 2 → 3 → 4
Consumer receives: 1 → 2 → 3 → 4  ✅ (guaranteed)
```

### 2. Throughput Limits
| Mode | Throughput |
|------|-----------|
| Without batching | **300 msg/s** |
| With batching | **3,000 msg/s** |

Compare to Standard queues: unlimited throughput. FIFO trades throughput for ordering.

### 3. Exactly-Once Send (Deduplication)
- Provide a **Deduplication ID** with each message
- If the same ID is seen twice within a **5-minute window**, the duplicate is removed
- Alternative: enable **content-based deduplication** at the queue level (hashes the message body)

### 4. Message Group ID
- **Required** for every message sent to a FIFO queue
- Ordering is guaranteed **within the same Message Group ID**
- Different group IDs can be processed in parallel (more on this in the next lesson)

### 5. Queue Naming
- FIFO queue name **must end with `.fifo`** (e.g., `MyQueue.fifo`)
- Won't let you create the queue without the suffix

## Exam Tips
- FIFO = **ordering guaranteed** + **limited throughput** (300/3,000 msg/s)
- Queue name must end with **`.fifo`**
- Deduplication window = **5 minutes**
- **Message Group ID** = required, defines ordering scope
- **Deduplication ID** = prevents duplicate messages within 5 min
- DLQ for FIFO must also be FIFO (from previous lesson)
