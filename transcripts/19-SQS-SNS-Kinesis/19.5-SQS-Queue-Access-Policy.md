# 19.5 - SQS Queue Access Policy

## Key Concepts

### 1. What Are SQS Queue Access Policies?
- JSON IAM **resource policies** attached directly to an SQS queue
- Similar to S3 Bucket Policies in structure and purpose
- Control who can send to or receive from a queue at the resource level (not the identity level)

### 2. Use Case 1 — Cross-Account Access
Account `444455556666` owns the queue. Account `111122223333` has the EC2 instance that needs to poll from it.

```json
{
  "Version": "2012-10-17",
  "Statement": [{
    "Effect": "Allow",
    "Principal": { "AWS": [ "111122223333" ] },
    "Action": [ "sqs:ReceiveMessage" ],
    "Resource": "arn:aws:sqs:us-east-1:444455556666:queue1"
  }]
}
```
- `Principal` = the **other account's ID** (the one that needs access)
- `Resource` = ARN of the queue in **the owner's account**
- `Action` = `sqs:ReceiveMessage`

### 3. Use Case 2 — S3 Event Notifications to SQS
Upload object to `bucket1` → S3 automatically sends a message to `queue1`.

```json
{
  "Version": "2012-10-17",
  "Statement": [{
    "Effect": "Allow",
    "Principal": { "AWS": "*" },
    "Action": [ "sqs:SendMessage" ],
    "Resource": "arn:aws:sqs:us-east-1:444455556666:queue1",
    "Condition": {
      "ArnLike": { "aws:SourceArn": "arn:aws:s3:*:*:bucket1" },
      "StringEquals": { "aws:SourceAccount": "<bucket1_owner_account_id>" }
    }
  }]
}
```
- `Principal` = `*` (any), but **locked down by the Condition**
- `Condition.ArnLike` = only this specific S3 bucket can trigger the send
- `Condition.SourceAccount` = only the bucket owner's account — prevents confused deputy attacks

## Exam Tips
- SQS Queue Access Policy = **resource policy**, JSON, similar to S3 bucket policy
- Two classic exam scenarios:
  - **Cross-account queue access** → Principal = the other account's ID
  - **S3 events → SQS** → Principal = `*` + `ArnLike` (bucket ARN) + `StringEquals` (source account)
- Without the access policy, S3 **cannot** write to the queue — IAM alone is not enough
- `SourceAccount` condition = security best practice to prevent confused deputy attacks
