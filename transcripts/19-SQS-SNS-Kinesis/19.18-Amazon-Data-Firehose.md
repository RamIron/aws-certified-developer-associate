# 19.18 - Amazon Data Firehose

## Key Concepts

### 1. What Is Amazon Data Firehose?
- Fully managed service to **load streaming data into destinations**
- Formerly called **Kinesis Data Firehose** (renamed because it does more than just Kinesis)
- Fully serverless, automatic scaling, pay for what you use
- Exam keyword: **near real-time** (not real-time like KDS)

### 2. Why "Near Real-Time"?
- Firehose uses a **buffer** that accumulates data before flushing
- Buffer flushes based on **size or time** thresholds
- This buffering delay makes it near real-time, not real-time

```
Records arrive → [Buffer accumulates...] → Flush to destination
                  (size or time trigger)
```

### 3. Sources (Producers)
- Your applications (SDK)
- Kinesis Agent
- **Kinesis Data Streams** (Firehose pulls from it)
- CloudWatch Logs & Events
- AWS IoT

### 4. Destinations

| Category | Destinations |
|----------|-------------|
| **AWS** | Amazon S3, Amazon Redshift, Amazon OpenSearch |
| **3rd Party** | Datadog, Splunk, New Relic, MongoDB |
| **Custom** | Any HTTP endpoint |

- Failed or all data can be backed up to an **S3 bucket**

### 5. Data Transformation
- Optional **Lambda function** for custom transformations (e.g., CSV → JSON)
- Built-in conversion to **Parquet** or **ORC** formats
- Compression: **gzip**, **snappy**
- Supported input: CSV, JSON, Parquet, Avro, text, binary

### 6. KDS vs Data Firehose — Critical Exam Comparison

| Feature | Kinesis Data Streams | Amazon Data Firehose |
|---------|---------------------|---------------------|
| **Purpose** | Streaming data collection | Load data into destinations |
| **Code** | Write your own producer/consumer | Fully managed, no code |
| **Latency** | **Real-time** | **Near real-time** (buffered) |
| **Scaling** | Provisioned or on-demand (manual shard mgmt) | Automatic |
| **Data storage** | Up to 365 days | **No storage** |
| **Replay** | ✅ Yes | ❌ No |
| **Destinations** | Your consumers process data | S3, Redshift, OpenSearch, 3rd party |

## Exam Tips
- **"Real-time"** → Kinesis Data Streams
- **"Near real-time"** → Amazon Data Firehose
- Firehose has **no data storage** and **no replay**
- Firehose is **fully managed** — no code, no capacity management
- **Lambda** can transform data inside Firehose before delivery
- Firehose can pull from **KDS**, CloudWatch, IoT
