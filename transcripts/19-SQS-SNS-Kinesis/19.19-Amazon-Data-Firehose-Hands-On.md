# 19.19 - Amazon Data Firehose - Hands On

## Exam-Relevant Observations

### 1. Sources (from Console)
- **Kinesis Data Streams** (used in demo)
- **Direct PUT:** Kinesis Agent, CloudWatch, IoT Core, EventBridge, SDK apps

### 2. Destinations (from Console)
- **AWS:** Amazon S3, Amazon OpenSearch (formerly ElasticSearch), Amazon Redshift
- **3rd Party:** Many services (don't need to memorize them)
- **Custom HTTP endpoint**
- ⚠️ Remember these three AWS destinations: **S3, OpenSearch, Redshift**

### 3. Buffer Settings (Key Exam Detail)
- Buffer flushes based on **size** OR **time**, whichever comes first:

| Setting | Default | Range |
|---------|---------|-------|
| Buffer size | 5 MB | 1–128 MB |
| Buffer interval | 300s (5 min) | **60s – 900s** (1–15 min) |

- Small buffer + short interval = faster delivery (less efficient)
- Large buffer + long interval = more efficient (slower delivery)
- Minimum interval = **60 seconds** — this is why Firehose is "near real-time"

### 4. Data Transformation
- Optional Lambda function to transform/filter/convert records before delivery
- Built-in record format conversion to **Parquet** or **ORC**
- Compression options: **GZIP, Snappy, Zip, Hadoop-Compatible Snappy**

### 5. IAM Role
- Firehose auto-creates an IAM role with permissions to:
  - **Read** from the Kinesis Data Stream source
  - **Write** to the S3 target bucket

### 6. Important: New Data Required
- Firehose only processes data sent **after** the delivery stream is created
- Historical data already in KDS is NOT picked up by Firehose

### 7. S3 Partitioning
- Data is automatically partitioned by **date** in S3 (year/month/day/hour)
