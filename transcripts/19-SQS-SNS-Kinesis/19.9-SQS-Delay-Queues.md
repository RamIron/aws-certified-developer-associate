# 19.9 - SQS Delay Queues

## Key Concepts

### 1. What Is a Delay Queue?
- Messages are **hidden from consumers** for a configurable period after being sent
- Default delay: **0 seconds** (messages available immediately)
- Maximum delay: **15 minutes**

```
Producer sends Msg → [Msg invisible for X seconds] → Consumer can poll it

Example (30s delay):
0s          30s
│            │
Send Msg     Msg becomes visible → Consumer polls ✅
```

### 2. Two Ways to Set the Delay
| Method | Scope | How |
|--------|-------|-----|
| **Queue-level default** | All messages in the queue | Set "Delivery delay" when creating/editing the queue |
| **Per-message override** | Single message | Use the `DelaySeconds` parameter when sending |

- Per-message `DelaySeconds` **overrides** the queue-level default

## Exam Tips
- Delay Queue max = **15 minutes**
- Default delay = **0 seconds** (no delay)
- `DelaySeconds` parameter = per-message override
- Don't confuse with **Visibility Timeout** (hides message *after* being polled) — Delay Queue hides message *before* first poll
