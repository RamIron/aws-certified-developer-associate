SQS Advanced ConceptsConceptos Avanzados SQS

Dead Letter Queue (DLQ)Cola de Mensajes Fallidos (DLQ)

After MaximumReceives threshold is exceeded, message moves to DLQ for debugging. DLQ type must match source (FIFO→FIFO, Standard→Standard). Set retention to 14 days. Use Redrive to Source to reprocess after fixing consumer. Cuando se excede el umbral MaximumReceives, el mensaje se mueve a la DLQ para depuracion. El tipo de DLQ debe coincidir (FIFO→FIFO, Standard→Standard). Retencion: 14 dias. Usar Redrive to Source para reprocesar despues de arreglar el consumidor.

Warning: If a message expires in the source queue before reaching MaximumReceives, it's lost forever — never reaches the DLQ. Atencion: Si un mensaje expira en la cola fuente antes de alcanzar MaximumReceives, se pierde para siempre — nunca llega a la DLQ.

Delay QueueCola con Retraso

Hide messages before first poll. Max 15 minutes. Default: 0s. Per-message override: DelaySeconds. Ocultar mensajes antes del primer poll. Max 15 minutos. Por defecto: 0s. Override por mensaje: DelaySeconds.

Long PollingLong Polling

Consumer waits for messages if queue is empty. Range: 1–20 seconds (20 recommended). Reduces API calls and cost. Enable via ReceiveMessageWaitTimeSeconds or queue-level setting. El consumidor espera mensajes si la cola esta vacia. Rango: 1–20 segundos (20 recomendado). Reduce llamadas API y costo. Activar via ReceiveMessageWaitTimeSeconds o a nivel de cola.

SQS Extended ClientCliente Extendido SQS

For messages > 1,024 KB: upload large payload to S3, send metadata pointer through SQS. Para mensajes > 1,024 KB: subir el payload grande a S3, enviar puntero de metadata por SQS.

ThroughputThroughput

Amazon SNS — Pub/SubAmazon SNS — Pub/Sub

Producer publishes to a topic. All subscribers receive the message. Data is not persisted — if not delivered, it's lost. El productor publica a un topic. Todos los suscriptores reciben el mensaje. Los datos no persisten — si no se entregan, se pierden.

SubscribersSuscriptores

SQS, Lambda, KDF, Email, Email-JSON, HTTP/HTTPS, SMSSQS, Lambda, KDF, Email, Email-JSON, HTTP/HTTPS, SMS

SNS FIFO TopicsTopics FIFO de SNS

Ordering + dedup. Subscribers: only SQS queues (Standard and FIFO). No Lambda, Email, HTTP. Orden + dedup. Suscriptores: solo colas SQS (Standard y FIFO). No Lambda, Email, HTTP.

Message FilteringFiltrado de Mensajes

JSON filter policy on the subscription (not the topic). No filter = receives all messages. Politica de filtro JSON en la suscripcion (no en el topic). Sin filtro = recibe todos los mensajes.

SNS + SQS Fan Out PatternPatron Fan Out SNS + SQS

Push once to SNS topic, multiple SQS queues subscribe. Decoupled, no data loss, cross-region supported. Publicar una vez al topic SNS, multiples colas SQS suscritas. Desacoplado, sin perdida de datos, soporte cross-region.

S3 event limitation: One rule per event type + prefix. To send same event to multiple destinations, use fan out: S3 → SNS → multiple SQS queues. Limitacion S3: Una regla por tipo de evento + prefijo. Para enviar el mismo evento a multiples destinos, usar fan out: S3 → SNS → multiples colas SQS.

SNS → KDF → S3 for persisting/archiving all messages flowing through a topic. SNS → KDF → S3 para persistir/archivar todos los mensajes que fluyen por un topic.
