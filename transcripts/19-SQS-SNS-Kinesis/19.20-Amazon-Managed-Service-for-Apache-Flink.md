# 19.20 - Amazon Managed Service for Apache Flink

## Why It Exists
- Process and transform data streams in **real-time** using Apache Flink framework
- Formerly called **Kinesis Data Analytics for Apache Flink**

## Who Uses It
- Teams needing complex real-time transformations on streaming data (beyond what Lambda or Firehose can do)

## What's on the Exam
- Can read from **Kinesis Data Streams** and **Amazon MSK** (Managed Streaming for Apache Kafka)
- ⚠️ **Cannot read from Amazon Data Firehose** (exam trick!)
- Uses **Java, SQL, or Scala** languages
- Fully managed: AWS provisions compute, auto-scaling, parallel computation
- Application backups via **checkpoints and snapshots**

## Simple Rule
- **Flink = real-time stream processing/transformation**
- KDS collects → Flink transforms → destination receives
- If the exam says "read from Firehose with Flink" → **wrong answer**
