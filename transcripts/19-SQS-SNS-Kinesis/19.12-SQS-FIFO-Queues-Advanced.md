# 19.12 - SQS FIFO Queues Advanced

## Key Concepts

### 1. Deduplication Deep Dive
- Deduplication interval: **5 minutes** (same message sent twice within 5 min → second is refused)
- Two methods:

| Method | How It Works |
|--------|-------------|
| **Content-based deduplication** | SQS computes a **SHA-256 hash** of the message body. Same body = same hash = duplicate refused |
| **Explicit Deduplication ID** | Producer provides a `MessageDeduplicationId` with each message. Same ID within 5 min = duplicate refused |

```
Content-based:
Producer sends "Hello World" → SHA-256 = abc123
Producer sends "Hello World" → SHA-256 = abc123 → ❌ duplicate refused
Producer sends "Hello World 2" → SHA-256 = xyz789 → ✅ different hash, accepted

Explicit ID:
Producer sends Msg (DedupID: "123") → ✅ accepted
Producer sends Msg (DedupID: "123") → ❌ refused (same ID within 5 min)
Producer sends Msg (DedupID: "456") → ✅ different ID, accepted
```

- If content-based deduplication is enabled, the Deduplication ID field becomes **optional**
- You can still override with an explicit ID even when content-based is enabled

### 2. Message Grouping (Parallel Processing)
- **Same Message Group ID** → all messages go to **one consumer**, strictly ordered
- **Different Message Group IDs** → each group gets its **own consumer**, enabling parallel processing
- Ordering is guaranteed **within** a group, **not across** groups

```
FIFO Queue:
┌─────────────────────────────────┐
│ Group "user123": A1 → A2 → A3  │──► Consumer 1 (ordered ✅)
│ Group "user234": B1 → B2 → B3  │──► Consumer 2 (ordered ✅)
│ Group "user345": C1 → C2       │──► Consumer 3 (ordered ✅)
└─────────────────────────────────┘
A1 vs B1 ordering? Not guaranteed ⚠️
```

**Typical use case:** Message Group ID = customer ID. Each customer's messages are processed in order, but different customers are processed in parallel. You can have **as many consumers as you have groups**.

## Exam Tips
- Deduplication interval = **5 minutes**
- Content-based dedup uses **SHA-256** hash of message body
- Same Message Group ID = **one consumer, strict order**
- Different Message Group IDs = **parallel consumers, order within each group**
- Want to scale FIFO consumers? Use **different Message Group IDs**
