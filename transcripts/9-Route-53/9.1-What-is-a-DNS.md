# 9.1 - What is a DNS?

## Key Concepts

### 1. What is DNS?
- **Domain Name System** вҖ” translates human-friendly hostnames into IP addresses
- The backbone of the internet вҖ” used every time you visit a website
- Hierarchical naming structure

### Key Terminology

| Term | What it is | Example |
|---|---|---|
| **Domain Registrar** | Where you register domain names | Route 53, GoDaddy |
| **DNS Records** | Entries that map hostnames to IPs | A, AAAA, CNAME, NS |
| **Zone File** | Contains all DNS records for a domain | вҖ” |
| **Name Server** | Server that resolves DNS queries | вҖ” |
| **TLD** | Top Level Domain | .com, .org, .gov |
| **SLD** | Second Level Domain | amazon.com, google.com |
| **FQDN** | Fully Qualified Domain Name | api.www.example.com |

- Local DNS server **recursively** queries more specific DNS servers
- Resolution order: **Local вҶ’ Root вҶ’ TLD вҶ’ SLD**
- Local DNS **caches** the result to avoid repeating the process
- The SLD server (e.g., Route 53) is where your DNS records live

---

## Exam Tips
- DNS = translates hostnames to IP addresses
- Know the hierarchy: Root вҶ’ TLD (.com) вҶ’ SLD (example.com) вҶ’ Subdomain (www.example.com)
- FQDN = Fully Qualified Domain Name (the complete hostname)
- Domain Registrar = where you register domains (Route 53, GoDaddy)
- Local DNS caches results for faster subsequent lookups
