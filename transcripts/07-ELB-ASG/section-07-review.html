Scalability & High Availability

📈

Vertical Scaling

Bigger instance (scale up / down). Limited by hardware ceiling. Example: t2.micro → t2.large.

🖥

Horizontal Scaling

More instances (scale out / in). No hardware limit. Enabled by ASG + ELB.

🏛

High Availability

Multi-AZ deployment — survives an AZ failure. Enabled by ASG multi-AZ + ELB multi-AZ.

2

Load Balancer Comparison

ALB — Application Load Balancer

Layer 7 · HTTP / HTTPS / WebSocket
  • Web apps, microservices, containers
  • Routing: URL path, hostname, query string, headers
  • No static IP — DNS name only
  • Targets: EC2, ECS, Lambda, private IPs
  • Real client IP via X-Forwarded-For header
Layer 7

NLB — Network Load Balancer

Layer 4 · TCP / UDP / TLS
  • Extreme performance, ultra-low latency
  • Static IPs per AZ (Elastic IP support)
  • Can be placed in front of ALB
  • Targets: EC2, private IPs, ALB
  • Sticky sessions via source IP (no cookies)
Layer 4

GWLB — Gateway Load Balancer

Layer 3 · IP Packets · GENEVE port 6081
  • Network traffic inspection (firewalls, IDS/IPS)
  • Routes every packet through 3rd-party appliances
  • Uses GENEVE protocol, port 6081
  • Targets: EC2, private IPs
  • Transparent to the application

Sticky Sessions

📌

What it does

Same client always routes to the same backend instance across requests.

🍪

ALB — Cookie-based

Application-based (custom cookie) or duration-based (LB-generated). Configured at target group level.

🌐

NLB — Source IP hash

No cookies — uses source IP to route consistently. Simpler but less flexible.

⚙️

Config level

Sticky sessions are configured at the TARGET GROUP level, not the listener or LB itself.

Reserved cookie names: AWSALB, AWSALBAPP, AWSALBTG — do not use these names in your own cookies.
⚠ Trade-off: Sticky sessions can cause load imbalance if some users generate significantly more load than others. A "heavy" user stays pinned to one instance, overloading it while others are idle.

SSL / TLS Certificates

🔐

SSL Termination

LB decrypts HTTPS traffic, then forwards plain HTTP to backend instances. Instances don't need to hold SSL certs.

📜

ACM — AWS Certificate Manager

Manages SSL/TLS certificates. Supports auto-renewal. Integrates natively with ALB and NLB.

🏷

SNI — Server Name Indication

Allows multiple SSL certs on a single load balancer listener. Supported by ALB & NLB only — Classic LB does NOT support SNI.

6

Connection Draining

💧

Also called

Deregistration Delay — same feature, different names depending on context.

⏳

What it does

In-flight requests get time to complete when an instance is deregistered or marked unhealthy. ELB stops sending NEW requests to the draining instance.

⚙️

Timing

Default: 300 seconds | Range: 1–3,600 s | Disable: set to 0

🎯

Best practice

Low value for short requests. High value for long uploads or connections. Set to 0 to deregister instantly.

7

Auto Scaling Groups (ASG)

📋

Launch Template

Blueprint for new EC2 instances — AMI, instance type, user data, security groups, key pair, etc. Never the running instance config.

📊

Min / Desired / Max

Capacity boundaries. Min: floor, Desired: target ASG tries to maintain, Max: ceiling for scaling out.

🔄

Self-Healing

Unhealthy instances are terminated and replaced automatically. New instances are registered with the ELB target group.

💸

ASGs are FREE

The ASG itself costs nothing. You only pay for the EC2 instances it launches.

⚠ Weak point — AMI Update
To change the AMI for future instances: update the Launch Template (not the running instances). Running instances keep their original config until they are terminated and replaced. The ASG does not automatically restart existing instances after a template change.
8

Scaling Policies

Target Tracking

Set a metric and a target value. ASG continuously adjusts capacity to keep the metric at target.

"Keep average CPU at 40%"

Simple / Step Scaling

CloudWatch alarm triggers a fixed change: add or remove N instances. Step scaling supports tiered thresholds.

"CPU > 70% → add 2 instances"

Scheduled Scaling

Scale at specific times for predictable patterns. Useful for known load spikes.

"Every Friday at 5pm → min 10"

Predictive Scaling

ML model analyzes historical CloudWatch data. Forecasts load ahead of time and schedules proactive scaling.

"Forecasts next week's traffic pattern"
Scaling Cooldown (default: 300 s) — After a scaling action, no new scaling activity occurs during the cooldown period. Gives metrics time to stabilize before the ASG makes another decision.
9

ASG Instance Refresh

Allows rolling replacement of all instances in an ASG — triggered via the StartInstanceRefresh API. Useful when you update the launch template with a new AMI and need all running instances replaced.

Instance Refresh — Rolling Replacement

⚠ Weak point — Warm-up time
Warm-up time does NOT auto-extend. If a new instance fails health checks during a refresh, the refresh PAUSES (because it cannot maintain the minimum healthy percentage). It does NOT extend the warm-up timer. The timer is fixed — once it expires, that instance is considered done regardless of health.

Exam Quick Reference

"Static IP / firewall whitelist" → NLB (Elastic IP per AZ)
"HTTP routing / microservices" → ALB
"Network inspection / GENEVE / port 6081" → GWLB
"Multiple SSL certs on one listener" → SNI — ALB & NLB only (not Classic LB)
"EC2 sees ___ IP from ALB" → ALB's private IP — use X-Forwarded-For for real client IP
"Cross-zone ON by default" → ALB only (free). NLB & GWLB off by default (charges apply)
"Sticky sessions on NLB" → Source IP hash (no cookies)
"Sticky sessions on ALB" → Cookies (application-based or duration-based)
"Connection draining default" → 300 seconds (range: 1–3,600 s, 0 = disabled)
"ASG launch template vs running instance" → Update launch template for NEW instances; running instances unaffected
"Instance Refresh trigger" → StartInstanceRefresh API
"Warm-up time behavior" → Fixed — does NOT auto-extend
"ASG self-healing" → Terminates unhealthy instances, replaces automatically
"Predictive scaling uses" → ML model, historical CloudWatch data, forecasts ahead
