# 17.10 - Beanstalk & CloudFormation

## Key Concepts

### 1. Beanstalk Uses CloudFormation Under the Hood

- Elastic Beanstalk is built on top of **CloudFormation**
- Every Beanstalk environment = a **CloudFormation stack** behind the scenes
- CloudFormation provisions all the AWS resources (ASG, ELB, EC2, security groups, etc.)
- You can see the stacks in the CloudFormation console — one stack per Beanstalk environment

### 2. What CloudFormation Creates for Each Environment

| Environment Type | Resources Created |
|-----------------|-------------------|
| **Single Instance (dev)** | ASG, Launch Config, Elastic IP, Security Group |
| **High Availability (prod)** | ASG, Launch Config, Scaling Policies, CloudWatch Alarms, Security Groups, ELB, Listener Rules, Target Group |

### 3. Why This Matters for EB Extensions

- Using CloudFormation resources in `.ebextensions/` = you can provision **anything**
- ElastiCache, DynamoDB, S3 buckets — anything CloudFormation supports
- This extends Beanstalk far beyond what the UI console allows

---

## Exam Tips

- Beanstalk uses **CloudFormation** under the hood to provision resources
- Each Beanstalk environment = one **CloudFormation stack**
- EB Extensions + CloudFormation = provision any AWS resource (ElastiCache, DynamoDB, S3, etc.)
- You don't need to manage CloudFormation directly — Beanstalk handles it
