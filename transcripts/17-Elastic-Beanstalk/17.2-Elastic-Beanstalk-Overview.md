# 17.2 - Elastic Beanstalk Overview (High Level)

## Key Concepts

### 1. What is Elastic Beanstalk?

- **Developer-centric** way to deploy applications on AWS
- Reuses components we already know: EC2, ASG, ELB, RDS, ElastiCache
- It's a **managed service** — handles provisioning, scaling, monitoring, configuration
- **You only worry about the code** — Beanstalk handles everything else
- Beanstalk itself is **free** — you pay for the underlying resources (EC2, RDS, etc.)

### 2. Beanstalk Components

| Component | What it is |
|-----------|-----------|
| **Application** | Collection of Beanstalk components (environments, versions, configs) |
| **Application Version** | An iteration of your code (v1, v2, v3...) |
| **Environment** | Collection of AWS resources running ONE application version |
| **Tier** | Type of environment: Web Server or Worker |

### 3. Workflow

```
1. Create Application
2. Upload a Version (your code)
3. Launch an Environment (resources are created)
4. Manage the environment lifecycle

To update: upload new version → deploy to environment
```

### 4. Web Server Tier vs Worker Tier

**Web Server Tier** (traditional):
```
Users → ELB → ASG → EC2 Instances (web servers)
```

**Worker Tier** (background processing):
```
SQS Queue → ASG → EC2 Instances (workers pull & process messages)
               ↑
  Scales based on number of SQS messages
```

- Can combine both: web tier pushes messages to worker tier's SQS queue
- Web tier = handles HTTP requests
- Worker tier = processes background jobs

### 5. Deployment Modes

| Mode | Architecture | Use case |
|------|-------------|----------|
| **Single Instance** | 1 EC2 + Elastic IP (+ optional RDS) | **Development** |
| **High Availability** | ELB + ASG + multi-AZ (+ optional RDS multi-AZ) | **Production** |

**Important:** RDS created **inside** Beanstalk = tied to environment lifecycle (delete env = delete DB). For production, create RDS **outside** Beanstalk.

### 6. Supported Languages

Go, Java SE, Java Tomcat, .NET Core (Linux), .NET (Windows), Node.js, PHP, Python, Ruby, Docker (single/multi/preconfigured), Packer Builder

---

## Exam Tips
- Beanstalk = **managed service**, developer only worries about **code**
- Beanstalk is **free** — you pay for underlying resources
- **Web Server Tier** = ELB + ASG (HTTP traffic)
- **Worker Tier** = SQS + ASG (background processing, scales on queue depth)
- **Single Instance** = dev, **High Availability with ELB** = production
- One environment = one application version at a time
- Supports almost any language + Docker
