# 17.11 - Beanstalk Cloning

## Key Concepts

### 1. What is Environment Cloning?

- Clone an existing Beanstalk environment into a **new environment** with the **exact same configuration**
- Use case: clone production → create a test/staging environment with identical settings

### 2. What Gets Preserved

| Preserved | NOT Preserved |
|-----------|---------------|
| Load balancer type and configuration | RDS **data** |
| RDS database **configuration** (type, size) | — |
| Environment variables | — |
| All other resource settings | — |

### 3. Workflow

1. Go to the environment → **Actions → Clone Environment**
2. Give the new environment a name (e.g., "test", "staging")
3. Optionally choose a new platform version
4. After cloning, customize settings via the **Configuration** tab

---

## Exam Tips

- Cloning preserves **all configuration** but **not RDS data**
- Used to create test/staging environments from production
- After cloning, settings can be modified independently
