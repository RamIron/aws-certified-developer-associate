# 17.9 - Beanstalk Extensions

## Key Concepts

### 1. What are EB Extensions?

- All parameters set in the **Beanstalk UI** can also be configured as **code**
- Allows infrastructure-as-code for Beanstalk configuration
- Useful for: environment variables, resource creation, option settings

### 2. Requirements

| Requirement | Detail |
|-------------|--------|
| **Directory** | Must be in `.ebextensions/` at the root of your source code |
| **Format** | YAML or JSON |
| **File extension** | Must end in `.config` (e.g., `logging.config`, `environment-variables.config`) |

### 3. What You Can Do with EB Extensions

- **Modify defaults** using `option_settings` (e.g., set environment variables)
- **Add resources** that can't be set via the console: RDS, ElastiCache, DynamoDB, etc.

Example `.ebextensions/environment-variables.config`:
```yaml
option_settings:
  aws:elasticbeanstalk:application:environment:
    DB_URL: "jdbc:postgresql://..."
    DB_USER: "username"
```

### 4. Important: Lifecycle of EB Extension Resources

- Resources created via EB extensions are **tied to the environment**
- If you **delete the environment** → those resources are **deleted too**
- Same rule as RDS created inside Beanstalk

---

## Exam Tips

- `.ebextensions/` = directory in root of source zip
- Files must end in `.config` and be YAML or JSON
- Use `option_settings` to set environment variables via code
- Can provision RDS, ElastiCache, DynamoDB through EB extensions
- Resources created by EB extensions are **deleted when the environment is deleted**
