# 12.4 - AWS CLI with MFA

## Key Concepts

### 1. Using MFA with CLI/SDK

- To use MFA with the CLI, you must create a **temporary session**
- The API to call is **STS GetSessionToken**
- This returns **temporary credentials** that prove MFA was used

### 2. How It Works

```
Step 1: Call STS GetSessionToken
        → pass --serial-number (MFA device ARN)
        → pass --token-code (code from MFA app)

Step 2: Receive temporary credentials
        → AccessKeyId
        → SecretAccessKey
        → SessionToken
        → Expiration (temporary!)

Step 3: Use those credentials for subsequent API calls
```

### 3. What You Get Back

| Field | Description |
|-------|------------|
| **AccessKeyId** | Temporary access key |
| **SecretAccessKey** | Temporary secret key |
| **SessionToken** | Token proving MFA was used |
| **Expiration** | When the credentials expire |

---

## Exam Tips
- **STS GetSessionToken** = the API to get temporary credentials with MFA
- Returns: AccessKeyId + SecretAccessKey + SessionToken (all temporary)
- You need the MFA device **ARN** (serial number) and the current **token code**
