# 12.8 - AWS Signature v4 Signing (Sigv4)

## Key Concepts

### 1. Why Sign Requests?

- Every HTTP API call to AWS must be **signed** so AWS knows who you are
- Signing uses your **AWS credentials** (access key + secret key)
- Exception: some requests don't need signing (e.g., reading a public S3 object)
- The CLI and SDK **sign requests automatically** — you don't do it manually

### 2. SigV4 (Signature Version 4)

- The signing protocol is called **SigV4**
- The process itself is complex (4 steps), but you do NOT need to know the steps
- You need to know the **two ways to transmit the signature**

### 3. Two Ways to Send the Signature

| Method | How | Example |
|--------|-----|---------|
| **Authorization Header** | Signature included in the HTTP `Authorization` header | What the CLI/SDK does by default |
| **Query String** | Signature included in the URL as `X-Amz-Signature` parameter | What S3 pre-signed URLs use |

---

## Exam Tips
- AWS API calls are signed using **SigV4**
- Two ways to transmit: **Authorization header** or **query string** (`X-Amz-Signature`)
- CLI and SDK sign automatically — you never do it manually
- S3 pre-signed URLs are an example of query string signing
