# 14.9 - S3 Access Logs

## Key Concepts

- Logs **all requests** to an S3 bucket (authorized or denied, any account)
- Logs are stored as files in **another S3 bucket** (must be in the same region)
- Can be analyzed with **Amazon Athena**

### Warning: Never Log to the Same Bucket!

```
❌ Bucket A → logs to Bucket A → logging loop → infinite growth → huge bill
✅ Bucket A → logs to Bucket B → no loop
```

---

## Exam Tips
- S3 Access Logs = audit all requests (authorized + denied)
- Logging bucket must be in the **same region**
- **NEVER** set the logging bucket to the monitored bucket (infinite loop)
- Analyze logs with **Amazon Athena**
