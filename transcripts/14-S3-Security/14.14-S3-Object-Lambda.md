# 14.14 - S3 Object Lambda

## Key Concepts

### 1. The Problem

You have one S3 bucket but different apps need different versions of the same object (redacted, enriched, transformed). You don't want to duplicate buckets.

### 2. The Solution: S3 Object Lambda

Use a **Lambda function** to modify the object **on the fly** as it's being retrieved, via an **S3 Object Lambda Access Point**.

```
E-commerce App → S3 Bucket (original data)

Analytics App → S3 Object Lambda Access Point
                    ↓
                Lambda (redact PII)
                    ↓
                S3 Access Point → S3 Bucket

Marketing App → S3 Object Lambda Access Point
                    ↓
                Lambda (enrich with loyalty DB) ← queries Customer Loyalty Database
                    ↓
                S3 Access Point → S3 Bucket
```

One bucket, multiple views of the data — each transformed by a different Lambda function.

### 3. How It Works

1. Caller accesses the **S3 Object Lambda Access Point**
2. This invokes a **Lambda function**
3. Lambda retrieves the original object from S3 (via a regular Access Point)
4. Lambda transforms the data (redact, enrich, convert, etc.)
5. Caller receives the modified object

### 4. Use Cases

- **Redact PII** (personally identifiable information) for analytics/non-prod
- **Convert formats** (XML → JSON)
- **Resize/watermark images** on the fly (watermark specific to the requesting user)
- **Enrich data** by looking up external databases
