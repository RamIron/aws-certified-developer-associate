# 14.11 - S3 Pre-signed URLs

## Key Concepts

### What Are Pre-signed URLs?

- A URL with **temporary credentials baked in** — whoever has the URL can access the file
- The user inherits the **permissions of the person who generated** the URL
- Works for both **GET** (download) and **PUT** (upload)

### Expiration

| Method | Max Expiration |
|--------|---------------|
| **S3 Console** | Up to 12 hours |
| **CLI** | Up to 168 hours (7 days) |

### How It Works

```
1. You (bucket owner) → generate pre-signed URL for a private file
2. S3 returns a URL with your credentials embedded
3. You share the URL with the target user
4. User accesses the file via the URL (no AWS account needed)
5. URL expires after the set time
```

### Use Cases

- Allow logged-in users to download a **premium video**
- Dynamically generate download URLs for an **ever-changing list of users**
- Temporarily allow a user to **upload a file** to a specific S3 location
- Keep the bucket **private** while granting temporary access

## Exam Tips
- Pre-signed URL = temporary access to a **private** S3 object
- User inherits the **generator's permissions**
- Console: max 12h | CLI: max 168h (7 days)

- "Temporary access to private S3 file" → pre-signed URL
