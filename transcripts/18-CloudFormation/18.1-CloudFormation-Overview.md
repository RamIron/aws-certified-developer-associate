# 18.1 - CloudFormation Overview

## Key Concepts

### What is CloudFormation?
- **Infrastructure as Code (IaC)** — declare your AWS infrastructure in a template file, CloudFormation creates it
- You define WHAT you want (declarative), CloudFormation figures out HOW and in what ORDER to create it
- Template = code (YAML or JSON) describing resources and their relationships

### Why use it?
- **No manual resource creation** — everything is code, version-controlled (Git)
- **Cost visibility** — all stack resources are tagged, cost is estimable
- **Productivity** — destroy and recreate infrastructure on the fly (e.g., delete dev at 5PM, recreate at 8AM)
- **Separation of concerns** — separate stacks for network, apps, databases
- **No reinventing the wheel** — reuse existing templates from the web/docs

### How it works
1. Templates uploaded to **Amazon S3**
2. CloudFormation references S3 to create a **Stack**
3. Stack = collection of AWS resources created/managed together
4. To update: re-upload a **new version** of the template (cannot edit the previous one)
5. Delete a stack → **all resources created by it are deleted**

### Deployment methods
| Method | How |
|--------|-----|
| **Manual** | Infrastructure Composer or code editor → Console (for learning) |
| **Automated** | Edit YAML → CLI or CD pipeline (recommended for production) |

### Template Building Blocks
| Component | Description | Required? |
|-----------|-------------|-----------|
| `AWSTemplateFormatVersion` | Template format version (internal to AWS) | No |
| `Description` | Comments about the template | No |
| `Resources` | AWS resources to create | **YES — only mandatory section** |
| `Parameters` | Dynamic inputs for the template | No |
| `Mappings` | Static variables | No |
| `Outputs` | References to what was created | No |
| `Conditions` | List of conditions for resource creation | No |

## Exam Tips
- CloudFormation = **Infrastructure as Code**
- Templates stored in **S3**, referenced by CloudFormation
- **Resources** = only mandatory section in a template
- Delete stack = **deletes all resources** created by it
- Cannot edit previous template — must **re-upload new version**
- Declarative = you don't specify order, CloudFormation handles orchestration
