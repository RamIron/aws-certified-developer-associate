# 18.6 - CloudFormation - Parameters

## Key Concepts

### 1. What Are Parameters and When to Use Them
- Parameters allow users to **provide inputs** into CloudFormation templates
- Use a parameter when:
  - The resource configuration is **likely to change in the future**
  - The value **cannot be determined ahead of time**
- Benefits: reuse templates across the company, prevent errors via type validation

### 2. Parameter Settings
Key settings you can define on a parameter:

| Setting | Purpose |
|---------|---------|
| `Type` | String, Number, CommaDelimitedList, List<Number>, AWS-Specific, SSM |
| `Description` | Human-readable label |
| `Default` | Value used if none provided |
| `AllowedValues` | List of valid values (creates a dropdown in console) |
| `AllowedPattern` | Regex validation |
| `MinLength` / `MaxLength` | String length constraints |
| `MinValue` / `MaxValue` | Numeric range constraints |
| `NoEcho` | Hides the value from logs (use for passwords/secrets) |
| `ConstraintDescription` | Message shown when validation fails |

### 3. AllowedValues and NoEcho Examples

**AllowedValues** — restricts to a fixed set of options:
```yaml
Parameters:
  InstanceType:
    Type: String
    Default: t2.micro
    AllowedValues:
      - t2.micro
      - t2.small
      - t2.medium
```

**NoEcho** — hides sensitive data from logs:
```yaml
Parameters:
  DBPassword:
    Type: String
    NoEcho: true
```

### 4. The !Ref Function
- Used to **reference parameters or resources** anywhere in the template
- Shorthand for `Fn::Ref`
- Works the same way for both parameters and resources

```yaml
# Referencing a parameter
GroupDescription: !Ref SecurityGroupDescription

# Referencing a resource
SecurityGroups:
  - !Ref SSHSecurityGroup
```

> ⚠️ Make sure parameters and resources don't share the same name — `!Ref` can't tell them apart.

### 5. Pseudo Parameters
- Built-in parameters provided by AWS — **no need to define them**
- Available by default in every template

| Reference | Returns |
|-----------|---------|
| `AWS::AccountId` | Your AWS account ID |
| `AWS::Region` | The region the stack is deployed in |
| `AWS::StackId` | The stack's ARN |
| `AWS::StackName` | The stack's name |
| `AWS::NotificationARNs` | List of SNS topic ARNs for stack notifications |
| `AWS::NoValue` | Removes a property (used in conditions) |

```yaml
# Example: use the current region dynamically
!Ref AWS::Region   # returns e.g. "us-east-1"
```

## Exam Tips
- **NoEcho: true** → for passwords and secrets (hides from logs)
- **AllowedValues** → creates a dropdown, restricts user input
- **!Ref** works for both parameters AND resources
- **Pseudo parameters** let your template self-discover account ID and region — no user input needed
