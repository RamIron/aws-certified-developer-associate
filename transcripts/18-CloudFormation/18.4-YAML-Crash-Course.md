# 18.4 - YAML Crash Course

## YAML Syntax Quick Reference

### Key-value pair
```yaml
invoice: 34843
date: "2001-01-23"
name: "Chris"
```

### Nested object (indentation = nesting)
```yaml
bill-to:
  given: Chris
  family: Dumars
  address:
    city: "Royal Oak"
```

### Array (dash = list item)
```yaml
products:
  - sku: BL394D
    quantity: 4
    description: Basketball
  - sku: BL4438H
    quantity: 1
    description: Super Hoop
```

### Multi-line string (pipe `|`)
```yaml
address-lines: |
  458 Walkman Dr.
  Suite #292
```

### Comments (hash `#`)
```yaml
# This is a comment
InstanceType: t2.micro  # inline comment
```

## In CloudFormation context
```yaml
Resources:           # nested object
  MyInstance:        # logical ID (nested object)
    Type: AWS::EC2::Instance   # key-value (string)
    Properties:      # nested object
      SecurityGroups:
        - !Ref SSHSecurityGroup   # list (dash) — first item
        - !Ref ServerSecurityGroup  # second item
```

## Why YAML over JSON?
- More readable
- Supports comments (`#`)
- No bracket/quote hell
- Preferred format in this course and in practice

## Exam relevance
Low — YAML syntax itself is not tested. But being able to **read** CloudFormation templates is essential for all remaining lessons.
