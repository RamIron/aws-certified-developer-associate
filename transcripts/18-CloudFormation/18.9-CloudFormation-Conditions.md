# 18.9 - CloudFormation - Conditions

## Key Concepts

### 1. What Are Conditions?
- Conditions control whether a **resource or output is created** based on a logical evaluation
- Common use cases:
  - Environment type (dev / test / prod)
  - AWS Region
  - A parameter value
- Conditions can reference each other, parameter values, or mappings

### 2. Defining a Condition
```yaml
Conditions:
  CreateProdResources: !Equals [!Ref EnvType, prod]
```
- `CreateProdResources` → condition name
- `!Equals` → compares two values; returns true if they match
- If `EnvType` parameter equals `"prod"` → condition is **true**

**Available condition functions:**

| Function | Description |
|----------|-------------|
| `!And` | True if all conditions are true |
| `!Or` | True if at least one condition is true |
| `!Not` | Inverts a condition |
| `!Equals` | True if two values are equal |
| `!If` | Returns one value if true, another if false |

### 3. Using a Condition on a Resource or Output
```yaml
Resources:
  MountPoint:
    Type: AWS::EC2::VolumeAttachment
    Condition: CreateProdResources   # only created if condition is true
    Properties:
      ...
```
- Add `Condition: <ConditionName>` to any resource or output
- If condition evaluates to **true** → resource is created
- If condition evaluates to **false** → resource is skipped

### 4. Visual Example
```
Parameters:
  EnvType: prod  ──► Condition: CreateProdResources = TRUE
                        └── MountPoint (EBS volume) → CREATED ✅

Parameters:
  EnvType: dev   ──► Condition: CreateProdResources = FALSE
                        └── MountPoint (EBS volume) → SKIPPED ❌
```

## Exam Tips
- You do NOT need to know how to write conditions from scratch — just know they exist and what they do
- Conditions can be applied to **resources and outputs**
- Common trigger: environment type parameter (`dev` / `test` / `prod`)
- Condition functions: `And`, `Or`, `Not`, `Equals`, `If`
