# 18.19 - CloudFormation - StackSets

## Key Concept

### What Are StackSets?
- Allow you to **create, update, or delete stacks across multiple AWS accounts and multiple regions** in a single operation
- Managed from a single **administrator account**
- When you update a StackSet → **all stack instances** in all target accounts and regions are updated simultaneously

### How It Works
```
Administrator Account
        ↓
Creates a StackSet (using a template)
        ↓
Deploys stack instances to:
  - Account A (us-east-1, eu-west-1)
  - Account B (us-east-1, ap-southeast-1)
  - Account C (us-east-1)
        ↓
Update StackSet → all instances updated at once ✅
```

### Common Use Case — AWS Organizations
- StackSets are commonly applied to **all accounts within an AWS Organization**
- Only an **administrator** (or a designated admin) can create StackSets
- Prevents chaos and security risks from unauthorized deployments

## Exam Tips
- StackSets = **deploy the same stack to multiple accounts and regions at once**
- Managed from a single **administrator account**
- Update once → **propagates to all stack instances automatically**
- Common with **AWS Organizations** to enforce baseline infrastructure across all accounts
