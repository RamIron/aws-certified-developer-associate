What is CloudFormation? ¿Qué es CloudFormation?

CloudFormation is AWS's Infrastructure as Code (IaC) service. You declare what resources you want in a YAML or JSON template, and CloudFormation creates them in the right order automatically. CloudFormation es el servicio de Infraestructura como Código (IaC) de AWS. Declarás qué recursos querés en una plantilla YAML o JSON, y CloudFormation los crea en el orden correcto de forma automática.

📄

DeclarativeDeclarativo

You define WHAT you want. CloudFormation figures out HOW and in what ORDER.Definís QUÉ querés. CloudFormation se encarga del CÓMO y en qué ORDEN.

🔁

Version ControlControl de Versiones

Templates are code — store them in Git, review changes, roll back to any version.Las plantillas son código: podés guardarlas en Git, revisar cambios y volver a versiones anteriores.

💸

Cost VisibilityVisibilidad de Costos

Resources are tagged automatically. Estimate costs and destroy/recreate environments on a schedule.Los recursos se etiquetan automáticamente. Estimá costos y destruí/recreá entornos según un horario.

🏗️

Separation of ConcernsSeparación de Concerns

Use separate stacks for networking, applications, and databases.Usá stacks separados para red, aplicaciones y bases de datos.

Key rules:Reglas clave: Templates must be uploaded to S3 before CloudFormation can use them — always, no exceptions. Even when you upload via the AWS console, AWS automatically uploads it to S3 behind the scenes. To update a stack you must re-upload a new version — you cannot edit the previous template in-place. Deleting a stack deletes all resources it created. Las plantillas deben subirse a S3 antes de que CloudFormation pueda usarlas — siempre, sin excepción. Incluso al subirlas desde la consola de AWS, AWS las sube a S3 automáticamente en segundo plano. Para actualizar un stack debés subir una nueva versión; no se puede editar la plantilla anterior en su lugar. Eliminar un stack elimina todos los recursos que creó.

Resources & Parameters

Resources

Resources are the core of every template. They represent the actual AWS components to create. CloudFormation supports 700+ resource types. Los recursos son el núcleo de toda plantilla. Representan los componentes de AWS a crear. CloudFormation soporta más de 700 tipos de recursos.

Type format:Formato de tipo: AWS::ServiceName::ResourceType
AWS::EC2::Instance  ·  AWS::Kinesis::Stream  ·  AWS::ElasticLoadBalancingV2::LoadBalancer

✅ Supported service?¿Servicio soportado?

Use a standard resource type. CloudFormation handles it natively.Usá un tipo de recurso estándar. CloudFormation lo maneja de forma nativa.

⚙️ Unsupported service?¿Servicio no soportado?

Use a Custom Resource backed by a Lambda function.Usá un Custom Resource respaldado por una función Lambda.

Parameters

Parameters let users provide inputs at deploy time. Use them when a value cannot be determined ahead of time or may change between deployments. Los parámetros permiten que los usuarios proporcionen entradas al momento del deploy. Usálos cuando un valor no se puede determinar de antemano o puede cambiar entre deploys.

Mappings · Outputs · Conditions

Mappings

Mappings hold fixed, hardcoded values defined in the template. Use them when you know all valid values in advance (e.g. AMI IDs per region). Los Mappings contienen valores fijos y codificados en duro definidos en la plantilla. Usalos cuando conocés todos los valores válidos de antemano (ej: IDs de AMI por región).

Mappings:
  RegionMap:
    us-east-1:
      HVM64: ami-0ff8a91507f77f867
    eu-west-1:
      HVM64: ami-047bb4163c506cd98

# Access with !FindInMap
# [MapName, TopLevelKey, SecondLevelKey]
ImageId: !FindInMap [RegionMap, !Ref "AWS::Region", HVM64]

Mappings

  • Values hardcoded in templateValores codificados en la plantilla
  • Finite, known in advanceFinitos, conocidos de antemano
  • Safer — you control all optionsMás seguro — controlás todas las opciones

Parameters

  • Values provided by user at deployValores proporcionados por el usuario al desplegar
  • Unknown ahead of timeDesconocidos de antemano
  • More flexible but less controlledMás flexible pero menos controlado

Outputs & Exports

The Outputs section declares values that can be viewed in the console and optionally exported for other stacks to import. La sección Outputs declara valores que se pueden ver en la consola y opcionalmente exportar para que otros stacks los importen.

Outputs:
  SSHSecurityGroup:
    Description: The SSH SG for our company
    Value: !Ref MySSHSecurityGroup
    Export:
      Name: SSHSecurityGroup # must be unique per region

# Import in another stack:
SecurityGroups:
  - !ImportValue SSHSecurityGroup
❌ Cannot delete a stack whose exports are still referenced by other stacks.No podés eliminar un stack cuyos exports siguen siendo referenciados por otros stacks.

Conditions

Conditions control whether a resource or output is created based on a logical evaluation — typically an environment parameter (dev / prod). Las Conditions controlan si un recurso o output se crea en función de una evaluación lógica — típicamente un parámetro de entorno (dev / prod).

# Define the condition
Conditions:
  CreateProdResources: !Equals [!Ref EnvType, prod]

# Apply it to a resource
Resources:
  MountPoint:
    Type: AWS::EC2::VolumeAttachment
    Condition: CreateProdResources # skipped if false

Service Role

An IAM Role you assign to CloudFormation so it can create/update/delete resources using its own permissions instead of yours. Un Rol IAM que le asignás a CloudFormation para que pueda crear/actualizar/eliminar recursos usando sus propios permisos en lugar de los tuyos.

👤

Without Service RoleSin Rol de Servicio

CloudFormation uses your personal IAM permissions. You must have direct access to all resources in the template.CloudFormation usa tus permisos IAM personales. Debés tener acceso directo a todos los recursos en la plantilla.

🔑

With Service RoleCon Rol de Servicio

CloudFormation uses the role's permissions. You only need cloudformation:* and iam:PassRole.CloudFormation usa los permisos del rol. Solo necesitás cloudformation:* y iam:PassRole.

🛡️

iam:PassRole

Required to assign a role to CloudFormation. Without it, the user cannot attach a service role to any stack.Requerido para asignar un rol a CloudFormation. Sin esto, el usuario no puede adjuntar un rol de servicio a ningún stack.

✅ Least privilege pattern:Patrón de mínimo privilegio: Users get cloudformation:* + iam:PassRole. The service role gets the actual resource permissions (S3, EC2, RDS…). Users never need those resource permissions directly. Los usuarios obtienen cloudformation:* + iam:PassRole. El rol de servicio obtiene los permisos reales de recursos (S3, EC2, RDS…). Los usuarios nunca necesitan esos permisos directamente.

Capabilities & Deletion PolicyCapabilities y Política de Eliminación

Capabilities

Explicit acknowledgements required when a template performs sensitive operations. Without them, CloudFormation throws InsufficientCapabilitiesException. Confirmaciones explícitas requeridas cuando una plantilla realiza operaciones sensibles. Sin ellas, CloudFormation lanza InsufficientCapabilitiesException.

CAPABILITY_IAM

Template creates/updates unnamed IAM resources (users, roles, groups, policies).La plantilla crea/actualiza recursos IAM sin nombre explícito.

CAPABILITY_NAMED_IAM

Template creates/updates IAM resources with an explicit name. Use this instead of IAM when names are set.La plantilla crea/actualiza recursos IAM con un nombre explícito. Usá este en lugar de IAM cuando los nombres están definidos.

CAPABILITY_AUTO_EXPAND

Template uses macros or nested stacks that expand before deployment.La plantilla usa macros o stacks anidados que se expanden antes del despliegue.

💡 Capabilities are just an acknowledgement — they don't grant extra permissions. Provide them via console checkbox, CLI --capabilities, or API. Las capabilities son solo una confirmación — no otorgan permisos extra. Proporcionálas via checkbox en la consola, --capabilities en CLI, o en la API.

Stack Policies & Termination ProtectionPolíticas de Stack y Protección contra Eliminación

Stack PoliciesPolíticas de Stack

A JSON document that controls which update actions are allowed on specific resources during a stack update. Protects against accidental changes to critical resources. Un documento JSON que controla qué acciones de actualización están permitidas en recursos específicos durante una actualización de stack. Protege contra cambios accidentales en recursos críticos.

{
  "Statement": [
    {
      "Effect": "Allow",  // allow updates on everything…
      "Action": "Update:*",
      "Principal": "*", "Resource": "*"
    },
    {
      "Effect": "Deny",   // …except the production DB
      "Action": "Update:*",
      "Principal": "*",
      "Resource": "LogicalResourceId/ProductionDatabase"
    }
  ]
}

Default (no policy)Predeterminado (sin política)

All resources can be updated freely.Todos los recursos se pueden actualizar libremente.

Once a policy is setUna vez que se establece una política

Default becomes Deny all. Must explicitly Allow what can be updated.El predeterminado se convierte en Denegar todo. Debés permitir explícitamente lo que puede actualizarse.

Deny vs AllowDeny vs Allow

A Deny always overrides an Allow. Same as standard IAM evaluation.Deny siempre tiene precedencia sobre Allow. Igual que la evaluación IAM estándar.

⚠ Stack policies only apply to stack updates — not to stack creation or deletion.Las políticas de stack solo aplican a actualizaciones — no a la creación o eliminación del stack.

Termination ProtectionProtección contra Eliminación

Prevents accidental deletion of an entire stack. Disabled by default — must be explicitly enabled. To delete a protected stack, disable protection first. Previene la eliminación accidental de un stack completo. Deshabilitada por defecto — debe activarse explícitamente. Para eliminar un stack protegido, primero deshabilitá la protección.

Custom Resources & StackSetsRecursos Personalizados y StackSets

Custom ResourcesRecursos Personalizados

Used when CloudFormation doesn't natively support a service, or you need custom provisioning logic (e.g. calling a third-party API, emptying a bucket before deletion). Se usan cuando CloudFormation no soporta nativamente un servicio, o cuando necesitás lógica de aprovisionamiento personalizada (ej: llamar a una API de terceros, vaciar un bucket antes de eliminarlo).

AWS::CloudFormation::CustomResource

  • Generic — works for any custom logicGenérico — funciona para cualquier lógica personalizada

Custom::MyCustomResourceTypeName

  • Recommended — more descriptive, same behaviorRecomendado — más descriptivo, mismo comportamiento
Resources:
  MyCustomResource:
    Type: Custom::EmptyS3BucketOnDelete
    Properties:
      ServiceToken: arn:aws:lambda:us-east-1:123:function:EmptyBucket
      # ServiceToken = Lambda ARN or SNS ARN (must be same region)
      BucketName: !Ref MyS3Bucket
