# 18.12 - CloudFormation - Service Role

## Key Concepts

### 1. What Is a CloudFormation Service Role?
- An **IAM Role** created specifically for CloudFormation
- Allows CloudFormation to **create, update, and delete stack resources** on your behalf
- Without a service role, CloudFormation uses **your own IAM permissions**
- With a service role, CloudFormation uses **the role's permissions** instead

### 2. Why Use a Service Role? (Least Privilege)
- Lets you grant users access to deploy stacks **without giving them direct permissions** on the underlying resources
- Example: a user can deploy a stack that creates S3 buckets, without having `s3:CreateBucket` themselves — CloudFormation uses the service role to do it

```
User permissions:
  - cloudformation:CreateStack ✅
  - iam:PassRole ✅
  - s3:CreateBucket ❌ (not needed!)

Service Role permissions:
  - s3:* ✅ (CloudFormation uses this to create the bucket)
```

### 3. The iam:PassRole Permission
- **Critical:** the user must have `iam:PassRole` to assign a service role to CloudFormation
- Without it, the user cannot attach the role to the stack
- `iam:PassRole` is a standard AWS permission required any time you pass a role to an AWS service

### 4. How It Works — End to End

```
1. Admin creates an IAM Role for CloudFormation (e.g. with S3 permissions)
2. Admin grants the user iam:PassRole + cloudformation:* permissions
3. User creates a stack and attaches the service role
4. CloudFormation uses the role's permissions to create/update/delete resources
5. User never needs direct permissions on the underlying resources
```

## Exam Tips
- Service roles = **least privilege** for CloudFormation deployments
- User needs **`iam:PassRole`** to pass a role to CloudFormation
- Without a service role → CloudFormation uses **your personal permissions**
- With a service role → CloudFormation uses **the role's permissions** (your personal permissions are irrelevant for resource creation)
- If a service role only has S3 permissions but the stack creates EC2 instances → **stack will fail**
