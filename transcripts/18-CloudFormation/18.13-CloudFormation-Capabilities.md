# 18.13 - CloudFormation - Capabilities

## Key Concepts

### 1. What Are Capabilities?
- Capabilities are **explicit acknowledgements** you must provide when your CloudFormation template performs certain sensitive operations
- They act as a safety gate — CloudFormation forces you to confirm you understand what the template will do
- Without the required capability → stack creation/update **fails** with `InsufficientCapabilitiesException`

### 2. The Three Capabilities

| Capability | When Required |
|-----------|--------------|
| `CAPABILITY_IAM` | Template creates/updates **unnamed** IAM resources (users, roles, groups, policies) |
| `CAPABILITY_NAMED_IAM` | Template creates/updates **named** IAM resources (with an explicit name property) |
| `CAPABILITY_AUTO_EXPAND` | Template uses **macros or nested stacks** (dynamic transformations before deployment) |

> Use `CAPABILITY_NAMED_IAM` instead of `CAPABILITY_IAM` when your IAM resources have explicit names — it's the stricter acknowledgement.

### 3. InsufficientCapabilitiesException
- Error thrown when a template **requires a capability** but you haven't acknowledged it
- Fix: re-submit the stack creation/update with the required capability

```
Submit stack → InsufficientCapabilitiesException
       ↓
Add CAPABILITY_NAMED_IAM (or other required capability)
       ↓
Resubmit → stack deploys successfully ✅
```

### 4. How to Provide Capabilities
- **AWS Console** → tick the acknowledgement checkbox at the bottom of the stack creation wizard
- **AWS CLI** → pass `--capabilities CAPABILITY_NAMED_IAM` as an argument
- **CloudFormation API** → include capabilities in the API call

## Exam Tips
- `CAPABILITY_NAMED_IAM` → IAM resources **with explicit names**
- `CAPABILITY_IAM` → IAM resources **without explicit names**
- `CAPABILITY_AUTO_EXPAND` → templates with **macros or nested stacks**
- `InsufficientCapabilitiesException` → you forgot to acknowledge a required capability
- Capabilities are just an acknowledgement — they don't grant extra permissions
