# 18.17 - CloudFormation - Custom Resources

## Key Concepts

### 1. What Are Custom Resources?
Used when you need to:
- Provision resources **not yet supported** by CloudFormation
- Run **custom provisioning logic** for on-premises or third-party resources
- Execute **custom scripts** during the create, update, or delete phases of a stack

Backed by either:
- **Lambda function** (most common) ⭐
- **SNS topic**

### 2. How to Define a Custom Resource

Two valid type formats:
- `AWS::CloudFormation::CustomResource` — the generic form
- `Custom::MyCustomResourceTypeName` — **recommended**, more descriptive

```yaml
Resources:
  MyCustomResource:
    Type: Custom::MyCustomResourceTypeName   # recommended
    Properties:
      ServiceToken: arn:aws:lambda:us-east-1:123456789012:function:MyLambdaFunction
      # ServiceToken = Lambda ARN or SNS ARN (must be in same region)
      InputParam1: value1   # optional input data passed to Lambda
      InputParam2: value2
```

- `ServiceToken` → the ARN of the Lambda function or SNS topic that handles the resource
- Additional properties → passed as input to the Lambda function

### 3. The Classic Use Case — Empty S3 Bucket Before Deletion ⭐
**Problem:** CloudFormation cannot delete a non-empty S3 bucket
**Solution:** Custom Resource backed by Lambda

```
Delete Stack triggered
      ↓
Custom Resource (Lambda) is invoked
      ↓
Lambda empties the S3 bucket (deletes all objects)
      ↓
CloudFormation deletes the now-empty S3 bucket ✅
```

This is a **common exam question** — remember this pattern!

### 4. Custom Resource Lifecycle
The Lambda function is invoked for each lifecycle event:
- **Create** → when the stack is created
- **Update** → when the custom resource properties change
- **Delete** → when the stack is deleted or the resource is removed

## Exam Tips
- Custom Resources = **Lambda-backed logic** for unsupported or custom provisioning
- Type can be `AWS::CloudFormation::CustomResource` OR `Custom::YourName` (recommended)
- `ServiceToken: <Lambda or SNS ARN>`
- **ServiceToken must be in the same region** as the stack
- Classic exam scenario: **empty an S3 bucket before stack deletion** → use a Custom Resource with Lambda
- Custom Resources can also provision **non-AWS resources** (on-premises, third-party)
