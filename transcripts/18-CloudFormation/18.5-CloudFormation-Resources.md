# 18.5 - CloudFormation - Resources

## Key Concepts

### 1. Resources - The Only Mandatory Section
- Resources are the **core** of every CloudFormation template
- They are the **only mandatory section** — everything else is optional
- Resources represent AWS components to be created and configured
- Resources can reference each other
- AWS handles creation, updates, and deletions automatically

### 2. Resource Type Identifier Format
```
service-provider::service-name::data-type-name
```
Examples:
- `AWS::EC2::Instance`
- `AWS::Kinesis::Stream`
- `AWS::ElasticLoadBalancingV2::LoadBalancer`

### 3. Over 700 Resource Types
- The list is always growing
- You don't need to memorize them — learn how to read the docs

### 4. How to Read the Documentation
- AWS CloudFormation resource docs list all available types
- Each resource page includes:
  - Syntax (JSON and YAML)
  - Properties (required vs optional)
  - Update behavior (No interruption / Replacement / etc.)
  - Return values
  - Examples
- Search tip: `"<resource name> cloudformation"` to find the right doc page

### 5. FAQ - Dynamic Number of Resources
- **Can I create a dynamic number of resources?**
  - Yes, but requires **CloudFormation Macros and Transform**
  - NOT in scope for DVA-C02
  - Everything written in the template = what gets created

### 6. FAQ - Is Every AWS Service Supported?
- **Almost** — only a few services are not yet supported
- Workaround: **CloudFormation Custom Resources**
- This IS on the exam — remember it!

## Exam Tips
- Resources is the **only mandatory section** in a CloudFormation template
- Resource type format: `AWS::ServiceName::ResourceType`
- If an AWS service is not natively supported → use **Custom Resources**
- Dynamic resource counts → **Macros and Transform** (not on DVA-C02 exam)
