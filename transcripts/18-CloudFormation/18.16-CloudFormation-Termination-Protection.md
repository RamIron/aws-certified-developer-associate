# 18.16 - CloudFormation - Termination Protection

## Key Concept

### What Is Termination Protection?
- A safety feature that **prevents accidental deletion** of a CloudFormation stack
- **Disabled by default** — must be explicitly enabled
- Can be toggled in the console, CLI, or API

### How It Works
```
Termination Protection ENABLED → Delete stack → ❌ BLOCKED
                                  (error: termination protection is enabled)
       ↓
Disable Termination Protection (requires IAM permissions)
       ↓
Delete stack → ✅ SUCCESS
```

### Stack Policy vs Termination Protection

| | Stack Policy | Termination Protection |
|--|-------------|----------------------|
| Protects against | Accidental **updates** to resources | Accidental **deletion** of the stack |
| Scope | Per-resource | Entire stack |
| Default | No policy (all updates allowed) | Disabled |

## Exam Tips
- Termination Protection = **delete protection** for the entire stack
- Default is **disabled** — must be explicitly turned on
- To delete a protected stack → first disable it, then delete
- Distinct from Stack Policy, which protects against **updates** not **deletes**
