# 7.11 - Elastic Load Balancer - Cross Zone Load Balancing

## Key Concepts

### 1. What is Cross Zone Load Balancing?

**With cross zone:** Each load balancer node distributes traffic evenly across **all registered instances in ALL AZs**.

**Without cross zone:** Each load balancer node distributes traffic only to instances **within its own AZ**.

### 2. Visual Example

```
With Cross Zone Load Balancing:

  Client ──► 50% ──► [ALB node AZ1] ──► 10% to each of ALL 10 instances
             50% ──► [ALB node AZ2] ──► 10% to each of ALL 10 instances

  AZ 1: 2 instances    AZ 2: 8 instances
  Each instance gets exactly 10% of total traffic ✅ (evenly distributed)


Without Cross Zone Load Balancing:

  Client ──► 50% ──► [ALB node AZ1] ──► only to AZ1 instances
             50% ──► [ALB node AZ2] ──► only to AZ2 instances

  AZ 1: 2 instances → each gets 25% of total traffic ⚠️
  AZ 2: 8 instances → each gets 6.25% of total traffic ⚠️
  (imbalanced!)
```

### 3. Default Settings & Charges Per Load Balancer

| Load Balancer | Cross Zone Default | Inter-AZ Data Charges |
|---------------|--------------------|-----------------------|
| ALB           | **ON** by default   | No charges            |
| NLB           | **OFF** by default  | Charges apply if enabled |
| GWLB          | **OFF** by default  | Charges apply if enabled |
| CLB           | **OFF** by default  | No charges            |

### 4. Configuration Level
- **ALB:** Enabled at load balancer level by default; can be disabled per **target group**
- **NLB / GWLB:** Configured at load balancer level (off by default)

---

## Exam Tips
- Cross zone = traffic distributed across ALL instances in ALL AZs
- Without cross zone = traffic stays within each AZ (can cause imbalance)
- **ALB:** ON by default, no extra charges
- **NLB / GWLB:** OFF by default, charges apply if enabled
- ALB cross zone can be overridden at the target group level
