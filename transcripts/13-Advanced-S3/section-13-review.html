Common Exam Scenario

Source images in /images/ prefix → Glacier after 60 days. Thumbnails in /thumbs/ prefix → One-Zone-IA → expire after 60 days. Use separate prefix-based rules for each.

Versioning + Lifecycle: Enable versioning to protect against accidental deletes. Then create lifecycle rules for non-current versions: Standard-IA after 30 days → Glacier Deep Archive after 90 days. This enables cost-effective recovery without keeping live copies in Standard storage.
S3 Analytics — Storage Class Analysis
What it does

Analyzes object access patterns and recommends when to transition from Standard to Standard-IA.

Limitations
  • Only Standard & Standard-IA
  • Does NOT work for One-Zone-IA
  • Does NOT work for Glacier tiers
Output & Timing

CSV report updated daily. Takes 24–48 hours to start generating data after enabling.

Key insight: There is no limit on the number of prefixes. If you spread objects across 4 prefixes, you get 4 × 5,500 = 22,000 GET/s total. Design your key naming strategy around this.
Upload Optimization
Feature What It Does Threshold
Multi-Part Upload Splits file into parts, uploads them in parallel, then reassembles. Speeds up large uploads and allows retry of individual parts. Recommended >100 MB
REQUIRED >5 GB
Transfer Acceleration Upload to the nearest AWS Edge Location, which then routes the data over AWS's private backbone to the target bucket region. Minimizes public internet latency. Best for long-distance uploads (e.g., US user uploading to ap-southeast-1). Compatible with Multi-Part Upload.
Download Optimization — Byte-Range Fetches
Parallel Downloads

Request specific byte ranges of an object simultaneously from multiple threads. Significantly improves download throughput for large files. If a range request fails, retry only that range — not the entire file.

Partial File Reads

Retrieve only the header of a file (first N bytes) without downloading the entire object. Useful for reading file metadata, magic bytes, or CSV headers stored in S3 at minimal cost.

Topic 4 — S3 Object Tags & Metadata
Attaching Attributes to S3 Objects
User-Defined Metadata
  • Key-value pairs set at upload time (in request headers)
  • Key must start with x-amz-meta-
  • Retrieved with the object via HEAD or GET request
  • Cannot be changed without re-uploading the object
  • Example: x-amz-meta-author: alice
Object Tags
  • Key-value pairs assignable after upload (can be changed)
  • Up to 10 tags per object
  • Used for fine-grained IAM permissions
  • Used for analytics and lifecycle rules
  • Example: department = finance
Using Tags for IAM Permissions
{
  "Condition": {
    "StringEquals": {
      "s3:ExistingObjectTag/department": "finance"
    }
  }
}
This condition restricts access to objects tagged department = finance only. Finance team IAM users get access; others are denied.
Critical Exam Point — Neither metadata nor tags are searchable in S3.
You cannot query S3 like a database. To find objects by metadata or tag values, you must build an external index in DynamoDB, query DynamoDB first to get the S3 key, then fetch the object from S3.

Exam Quick-Fire Tips
Scenario: "Lifecycle rule types"
→ Transition (move to cheaper class) and Expiration (delete)
Scenario: "S3 Analytics recommendation"
→ Only for Standard & Standard-IA. CSV report. Takes 24–48h to start.
Scenario: "Trigger Lambda when object uploaded to S3"
→ S3 Event Notification → Lambda. Lambda needs a resource policy (not IAM role).
Scenario: "S3 event to SNS or SQS"
→ Set a resource-based policy on the SNS topic / SQS queue to allow S3 to publish.
Scenario: "S3 event to multiple destinations simultaneously"
→ Use EventBridge (routes to 18+ AWS services, advanced filtering, archiving).
Scenario: "S3 performance baseline"
→ 3,500 PUT/s and 5,500 GET/s per prefix. Multiply by spreading prefixes.
Scenario: "Upload a 10 GB file to S3"
→ Multi-Part Upload is REQUIRED for files over 5 GB (recommended over 100 MB).
Scenario: "User in Australia uploading to S3 bucket in eu-west-1"
→ Transfer Acceleration — uploads to nearby edge, private AWS backbone to bucket.
Scenario: "Download large file faster / read only first 100 bytes"
→ Byte-Range Fetches — parallel range requests or partial file retrieval.
Scenario: "Different IAM permissions per S3 object"
→ Apply object tags, then use IAM condition s3:ExistingObjectTag/key = value.
Scenario: "Search S3 objects by metadata or tag"
→ NOT possible natively in S3. Must build DynamoDB index and query there first.
Scenario: "S3 Analytics does NOT work for…"
→ One-Zone-IA and any Glacier tier. Only Standard ↔ Standard-IA analysis.
