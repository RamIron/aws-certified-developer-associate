# 13.6 - S3 Object Tags & Metadata

## Key Concepts

### 1. User-Defined Object Metadata

- Key-value pairs attached to objects at upload time
- User-defined metadata names must start with **`x-amz-meta-`**
- AWS also adds its own metadata (e.g., `Content-Length`, `Content-Type`)
- Retrieved when you retrieve the object

### 2. S3 Object Tags

- Key-value pairs (like AWS tags on any resource)
- Used for:
  - **Fine-grained permissions** — IAM policies can grant access based on tags
  - **Analytics** — S3 Analytics can group by tags

### 3. Metadata vs Tags

| Feature | Metadata | Tags |
|---------|----------|------|
| Format | `x-amz-meta-key: value` | `Key: Value` |
| Use case | Info about the object | Permissions & analytics |
| Searchable in S3? | **NO** | **NO** |

### 4. Searching S3 Objects — The DynamoDB Pattern

**Neither metadata nor tags are searchable in S3.**

To search S3 objects by metadata/tags, use an external index:

```
S3 Objects (metadata + tags)
    ↓ sync to
DynamoDB (searchable index)
    ↓ query
Results → fetch objects from S3
```

## Exam Tips
- User metadata prefix: **`x-amz-meta-`**
- Tags → fine-grained permissions + analytics
- **Metadata and tags are NOT searchable** in S3
- "Search S3 objects by metadata" → build an index in **DynamoDB**
