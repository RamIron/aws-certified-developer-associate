# 13.1 - S3 Lifecycle Rules (with S3 Analytics)

## Key Concepts

### 1. Moving Between Storage Classes

Objects can be transitioned between classes following this flow:

```
Standard → Standard-IA → Intelligent-Tiering → One-Zone-IA
                                                     ↓
                                              Glacier Flexible → Glacier Deep Archive
```

You can move objects **manually** or **automate it with Lifecycle Rules**.

### 2. Lifecycle Rules — Two Types of Actions

| Action | What it does | Example |
|--------|-------------|---------|
| **Transition Actions** | Move objects to another storage class | Move to Standard-IA after 60 days, to Glacier after 6 months |
| **Expiration Actions** | Delete objects after some time | Delete access logs after 365 days, delete old versions, delete incomplete multi-part uploads |

### 3. Rule Scope

Rules can be scoped by:
- **Entire bucket** — applies to all objects
- **Prefix** — only objects in a specific path (e.g., `thumbnails/`)
- **Object tags** — only objects with specific tags (e.g., `department:finance`)

### 4. Exam Scenarios

**Scenario 1 — Source images + thumbnails:**
- Source images: Standard → transition to Glacier after 60 days (user can wait 6 hours after that)
- Thumbnails: One-Zone-IA → expire/delete after 60 days (easily recreated, infrequently accessed)
- Use **prefix** to differentiate (`source/` vs `thumbnails/`)

**Scenario 2 — Recover deleted objects:**
- Enable **versioning** (delete = hidden by delete marker, recoverable)
- Non-current versions → transition to **Standard-IA** (recover within 30 days)
- Then transition non-current versions → **Glacier Deep Archive** (recover within 48 hours, up to 365 days)

### 5. S3 Analytics

- Helps determine the **optimal number of days** to transition objects between classes
- Works for **Standard** and **Standard-IA** only (NOT One-Zone-IA or Glacier)
- Produces a **CSV report** updated daily
- Takes **24-48 hours** to start showing data
- Good first step to create or improve lifecycle rules

---

## Exam Tips
- Lifecycle Rules automate transitions between storage classes and deletions
- **Transition actions** = move | **Expiration actions** = delete
- Rules can target by prefix or object tags
- S3 Analytics helps decide when to transition (Standard & Standard-IA only)
- "Recover deleted objects" → versioning + lifecycle rules on non-current versions
- "Easily recreated" + "infrequently accessed" → One-Zone-IA
