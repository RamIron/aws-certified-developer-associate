# 8.11 - Amazon MemoryDB for Redis - Overview

## Key Concepts

### 1. What is MemoryDB for Redis?
- A **Redis-compatible, durable, in-memory database** service
- NOT a cache — it's a **primary database** with a Redis-compatible API
- Ultra-fast: **160M+ requests/sec**
- **Durable** with Multi-AZ transaction log
- Scales from tens of GB to hundreds of TB
- Use cases: microservices, web/mobile apps, online gaming, media streaming

### ElastiCache Redis vs MemoryDB for Redis

| | ElastiCache (Redis) | MemoryDB for Redis |
|---|---|---|
| **Primary purpose** | **Cache** with some durability | **Database** with Redis-compatible API |
| **Use it as** | Cache layer in front of another DB | Primary database on its own |
| **Durability** | Data can be lost (it's a cache) | Durable — Multi-AZ transaction log |
| **Performance** | High | Ultra-fast (160M+ requests/sec) |
| **Storage scale** | Limited | Tens of GB to hundreds of TB |

```
ElastiCache:                         MemoryDB:

[App] → [ElastiCache] → [RDS]       [App] → [MemoryDB for Redis]
         (cache layer)                        (the database itself)
```

## Exam Tips
- ElastiCache Redis = **cache** in front of your database
- MemoryDB for Redis = **is** your database (with Redis API)
- See "durable in-memory database" + "Redis-compatible" → **MemoryDB**
- See "reduce read load" + "cache" → **ElastiCache**
- MemoryDB: Multi-AZ transaction log for durability and fast recovery
