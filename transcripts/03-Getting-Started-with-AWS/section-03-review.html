AWS Global Infrastructure

AWS is distributed across the globe in a layered hierarchy: Regions contain Availability Zones, which contain data centers. On top of this, Edge Locations extend reach for content delivery.

🗺

Region

A cluster of data centers in a specific geographic area. Named like us-east-1, eu-west-3. AWS has 30+ regions globally.

🏢

Availability Zone (AZ)

One or more discrete data centers within a region, each with redundant power, networking, and connectivity. Typically 3 per region (range: 3–6).

🛡

AZ Isolation

AZs are physically isolated from each other — separated by meaningful distance to prevent disaster cascading across AZs.

⚡

AZ Connectivity

Despite isolation, AZs within a region are connected via high-bandwidth, ultra-low-latency private networking — fast enough to act as one logical unit.

How to Choose a Region

Not all regions are equal. Evaluate these four factors in priority order when deciding where to deploy your workload:

1

Compliance

Data governance and legal requirements come first. Some regulations require data to stay within a specific country (e.g., GDPR in Europe). This is non-negotiable.

2

Latency

Deploy close to your users. If most users are in Brazil, choose sa-east-1. Less distance = fewer milliseconds = better UX.

3

Service Availability

Not all AWS services are available in all regions. Newer services often launch in us-east-1 first. Verify the service you need exists in your chosen region.

4

Pricing

AWS pricing varies by region. The same EC2 instance type can cost differently in us-east-1 vs ap-southeast-1. Factor in costs last, after compliance and UX.

⚠ On the exam, if a question asks what to consider first when choosing a region — the answer is Compliance, not latency or cost.

Global vs Regional Services

Some AWS services operate globally (no region selection needed), while most are regional (you deploy per region).

🌐 Global Services

  • IAM — Identity and Access Management
  • Route 53 — DNS service
  • CloudFront — Content Delivery Network
  • WAF — Web Application Firewall

📍 Regional Services

  • EC2 — Compute instances
  • Lambda — Serverless functions
  • Elastic Beanstalk — App deployment
  • RDS — Relational databases
  • S3 — Object storage (bucket is regional)
  • DynamoDB — NoSQL database
  • ECS / EKS — Container services
  • VPC — Virtual Private Cloud
S3 nuance: S3 bucket names are globally unique, but the bucket itself is created in a specific region. The S3 console may appear global but data lives regionally.

Exam Tips

"IAM is a ___ service" → Global (no region selection)
"Region =" → Cluster of data centers
"AZs are ___ from each other" → Isolated — prevents disaster cascading
"AZs are connected with" → High-bandwidth, ultra-low-latency networking
"Edge locations are used by" → CloudFront (CDN)
"How many AZs per region" → 3 to 6 (usually 3)
"Choose region based on" → Compliance > Latency > Service availability > Pricing
"How many Edge Locations exist" → 400+ PoPs in 90+ cities, 40+ countries
