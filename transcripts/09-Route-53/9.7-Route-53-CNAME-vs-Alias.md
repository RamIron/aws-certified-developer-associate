# 9.7 - Route 53 - CNAME vs Alias

## Key Concepts

### 1. The Problem
AWS resources (ALB, CloudFront, etc.) expose a hostname like `my-alb-123.us-east-1.elb.amazonaws.com`. You want to map your own domain (`myapp.example.com`) to it.

Two options: **CNAME** or **Alias**.

### 2. CNAME vs Alias Comparison

| Feature | CNAME | Alias |
|---------|-------|-------|
| **Points to** | Any hostname | AWS resources only |
| **Root domain (zone apex)?** | ❌ Not allowed | ✅ Works |
| **Cost** | Charged per query | **Free** |
| **Health checks** | No | Yes (native) |
| **TTL** | You set it | Set automatically by Route 53 |
| **Record type** | CNAME | A or AAAA |

### 3. Zone Apex (Root Domain) — Critical for Exam

```
example.com         → CNAME ❌ (not allowed — "apex of the zone")
example.com         → Alias  ✅ (works!)
www.example.com     → CNAME ✅ (works — it's a subdomain)
www.example.com     → Alias  ✅ (works too)
```

If the exam asks about pointing a **root domain** to an AWS resource → the answer is **Alias record**.

### 4. Alias Record Targets (AWS Resources Only)

| Supported Targets | |
|---|---|
| Elastic Load Balancers (ALB/NLB) | ✅ |
| CloudFront Distributions | ✅ |
| API Gateway | ✅ |
| Elastic Beanstalk environments | ✅ |
| S3 Websites (not regular S3 buckets) | ✅ |
| VPC Interface Endpoints | ✅ |
| Global Accelerator | ✅ |
| Route 53 records (same hosted zone) | ✅ |
| **EC2 DNS name** | ❌ **NOT supported** |

### 5. Alias Record Details
- Always of type **A** (IPv4) or **AAAA** (IPv6)
- If the underlying resource's IP changes, the alias **automatically recognizes** the new IP
- TTL is **managed by Route 53** — you cannot set it manually

---

## Exam Tips
- **CNAME** cannot be used for zone apex (root domain) — **Alias** can
- Alias records are **free** and have **native health checks**
- Alias targets: ELB, CloudFront, API Gateway, Beanstalk, S3 Websites, VPC Endpoints, Global Accelerator, Route 53 records
- **EC2 DNS name is NOT a valid Alias target** — must remember this
- Alias is always type A or AAAA, TTL set automatically
- Root domain + AWS resource → always Alias
