# 9.2 - Route 53 Overview

## Key Concepts

### 1. What is Route 53?
- **Highly available, scalable, fully managed** DNS service
- **Authoritative** — you have full control to update DNS records
- Also a **Domain Registrar** — you can buy domain names directly
- Can check **health of resources**
- Only AWS service with **100% availability SLA**
- Named after port **53** — the traditional DNS port

```
┌─────────┐  example.com?   ┌─────────────┐  A: 54.22.33.44  ┌─────────┐
│ Client  │─────────────────►│  Route 53   │◄────────────────│ Hosted  │
│         │◄─────────────────│             │                  │  Zone   │
│         │  54.22.33.44     └─────────────┘                  └─────────┘
└────┬────┘
     │ connects
     ▼
┌─────────┐
│   EC2   │ (54.22.33.44)
└─────────┘
```

### What is an SLA?

**SLA (Service Level Agreement)** = a formal guarantee from AWS about service uptime. If AWS fails to meet it, you get service credits (money back).

| SLA | Max downtime per year | Example services |
|---|---|---|
| **99.9%** | ~8.7 hours | — |
| **99.99%** | ~52 minutes | Most AWS services |
| **100%** | Zero downtime guaranteed | **Route 53 only** |

Route 53 is the **only** AWS service that offers a 100% availability SLA. This is a key exam fact.

### Cost
- **$0.50/month** per hosted zone
- **$12+/year** for domain name registration

### 2. DNS Records

Each DNS record contains:

| Field | Description | Example |
|---|---|---|
| **Domain/Subdomain** | The hostname | example.com |
| **Record Type** | Type of record | A, AAAA, CNAME, NS |
| **Value** | The target IP or hostname | 12.34.56.78 |
| **Routing Policy** | How Route 53 responds to queries | Simple, Weighted, etc. |
| **TTL** | How long the record is cached | 300 seconds |

### Record Types (must know for exam)

| Type | Maps to | Example |
|---|---|---|
| **A** | Hostname → **IPv4** | example.com → 1.2.3.4 |
| **AAAA** | Hostname → **IPv6** | example.com → 2001:db8::1 |
| **CNAME** | Hostname → **another hostname** | app.example.com → myapp.elb.amazonaws.com |
| **NS** | **Name servers** for the hosted zone | Controls how traffic is routed to a domain |

### CNAME restriction (important for exam)
- You **cannot** create a CNAME for the **zone apex** (root domain)
- example.com → CNAME ❌ (not allowed)
- www.example.com → CNAME ✅ (allowed)
- Solution: use **Alias records** (covered in lesson 9.7)

### 3. Hosted Zones — Public vs Private

| | Public Hosted Zone | Private Hosted Zone |
|---|---|---|
| **Resolves** | Public domain names | Private domain names |
| **Accessible from** | Anyone on the internet | Only within your VPC(s) |
| **Example domain** | example.com | company.internal |
| **Use case** | Public websites, APIs | Internal services, databases |

---

## Exam Tips
- Route 53 = managed DNS + domain registrar
- **Only AWS service with 100% availability SLA**
- Port 53 = DNS port (hence the name)
- Know record types: A (IPv4), AAAA (IPv6), CNAME (hostname → hostname), NS (name servers)
- **CNAME cannot be used for zone apex** (root domain) — use Alias instead
- Public hosted zone = internet-facing, Private hosted zone = VPC only
- $0.50/month per hosted zone, $12+/year for domain registration
