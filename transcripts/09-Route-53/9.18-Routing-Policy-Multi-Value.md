# 9.18 - Routing Policy - Multi Value

## Key Concepts

### 1. How Multi-Value Routing Works

- Returns **multiple records** (up to 8) in a single DNS query
- Can be associated with **health checks** — only healthy records are returned
- The **client picks one** of the returned records (client-side load balancing)
- **Not a substitute for ELB** — it's DNS-level, not true load balancing

```
Client ──► Route 53 (Multi-Value query)
               │
               ├──► EC2 us-east-1     ✅ healthy → included
               ├──► EC2 ap-southeast-1 ✅ healthy → included
               └──► EC2 eu-central-1   ❌ unhealthy → excluded

Response: [us-east-1 IP, ap-southeast-1 IP]  (2 of 3 returned)
Client picks one randomly
```

### 2. Multi-Value vs Simple Routing

| | Simple | Multi-Value |
|--|--------|------------|
| **Multiple values** | Yes (in one record) | Yes (separate records) |
| **Health checks** | **No** | **Yes** |
| **Unhealthy resources** | Can be returned | Excluded from response |
| **Max records returned** | All values in the record | Up to **8 healthy** records |

---

## Exam Tips
- Multi-Value = multiple records + health checks → only healthy ones returned
- Up to **8 healthy records** per query
- **Not a replacement for ELB** — it's client-side selection, not true load balancing
- Key difference from Simple: Multi-Value supports **health checks**, Simple does not
- Exam signal: "return multiple IPs but exclude unhealthy ones" → **Multi-Value**
