# 9.9 - Routing Policy - Weighted

## Key Concepts

### 1. How Weighted Routing Works
- Assign a **relative weight** to each record
- Traffic percentage = weight of record / sum of all weights
- Weights **don't need to sum to 100** — they're relative
- All records must have the **same name and type**
- **Supports health checks**

```
                         weight 70 (70%)
Route 53 ────────────────────────────────► EC2 (us-east-1)
    │
    │                    weight 20 (20%)
    ├────────────────────────────────────► EC2 (eu-central-1)
    │
    │                    weight 10 (10%)
    └────────────────────────────────────► EC2 (ap-southeast-1)
```

### 2. Special Cases
- **Weight = 0** → stop sending traffic to that resource
- **All weights = 0** → all records returned with equal weight

### 3. Use Cases
- **Load balancing** across regions
- **Testing a new app version** — send a small % of traffic to it (e.g., weight 5 out of 100)
- **Gradual migration** — shift weight from old to new over time

### 4. Weighted vs Simple
| | Simple | Weighted |
|---|---|---|
| Multiple IPs | All returned, client picks randomly | One returned based on weight |
| Health checks | ❌ | ✅ |
| Control | No control over distribution | Fine-grained % control |
