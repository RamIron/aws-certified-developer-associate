# 9.8 - Routing Policy - Simple

## Key Concepts

### 1. DNS Routing vs Load Balancer Routing
- DNS routing is **NOT** traffic routing — DNS doesn't see or handle traffic
- DNS just **answers queries** — tells the client which IP to connect to
- The client then connects directly to the resource

### 2. Route 53 Routing Policies Overview
| Policy | Purpose |
|--------|---------|
| **Simple** | Route to a single resource (or random from multiple) |
| **Weighted** | Distribute traffic by percentage |
| **Failover** | Active-passive failover |
| **Latency-based** | Route to lowest latency region |
| **Geolocation** | Route based on user's location |
| **Multi-value answer** | Return multiple healthy IPs |
| **Geoproximity** | Route based on geographic distance with bias |

### 3. Simple Routing Policy
- Routes to a **single resource** typically
- Can specify **multiple values** in the same record → client picks one **randomly**
- If using Alias → can only specify **one AWS resource**
- **Cannot** associate with health checks

```
Client ──► "foo.example.com?" ──► Route 53
Client ◄── "A: 11.22.33.44, 55.66.77.88" ◄── Route 53
                │
                └── Client picks one randomly
```

---

## Exam Tips
- DNS routing ≠ traffic routing — DNS only answers queries, doesn't handle traffic
- Simple policy: can return multiple IPs, client picks randomly
- Simple policy: **no health checks** supported
- Simple + Alias: only one AWS resource allowed
- Know all 7 routing policies by name
