# 9.13 - Routing Policy - Failover

## Key Concepts

### 1. How Failover Routing Works

- Uses **two records**: one **Primary**, one **Secondary** (disaster recovery)
- Health check on the primary is **mandatory**
- Health check on the secondary is **optional**
- Only **one primary** and **one secondary** allowed

```
                         Health Check ✅
                              │
Client ──► Route 53 ──► [Primary EC2] ← healthy → returns primary IP
                              │
                         Health Check ❌
                              │
Client ──► Route 53 ──► [Secondary EC2] ← failover → returns secondary IP
```

### 2. Automatic Failover Behavior

- When primary health check = **healthy** → Route 53 returns primary record
- When primary health check = **unhealthy** → Route 53 automatically returns secondary record
- Failover is **seamless** — the client just gets a different IP
- When primary recovers → Route 53 **fails back** to primary automatically

---

## Exam Tips
- Failover = **active-passive** setup (primary + secondary)
- Health check on primary is **mandatory**
- Health check on secondary is **optional**
- Only **2 records** allowed (1 primary + 1 secondary)
- Automatic failover AND automatic fail-back when primary recovers
