# 9.17 - Routing Policy - IP-based

## Key Concepts

### 1. How IP-based Routing Works

- You define a list of **CIDR blocks** (IP ranges) in Route 53
- Each CIDR block is mapped to a **location**
- Each location is linked to a specific **record/endpoint**
- Route 53 checks the client's IP → matches it to a CIDR → returns the corresponding record

```
Route 53 CIDR locations:
  Location 1: 203.0.0.0/16 ──► 1.2.3.4 (EC2 instance A)
  Location 2: 200.0.0.0/16 ──► 5.6.7.8 (EC2 instance B)

User (IP: 203.0.5.10) ──► matches Location 1 ──► gets 1.2.3.4
User (IP: 200.0.1.50) ──► matches Location 2 ──► gets 5.6.7.8
```

### 2. Use Cases

- **Optimize performance** — you know where specific IPs come from
- **Reduce network costs** — route known ISP traffic to the nearest endpoint
- **ISP-specific routing** — a specific internet provider uses a known CIDR block, route them to a dedicated endpoint

---

## Exam Tips
- IP-based = route by **client's IP address** using CIDR blocks
- You define the CIDR-to-location mapping yourself (not automatic)
- Key signal: "known IP ranges", "specific ISP", "CIDR-based routing" → **IP-based**
- Different from Geolocation (country/continent) — IP-based uses actual IP addresses, not geographic lookup
