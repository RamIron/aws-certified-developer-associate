# 20.19 - X-Ray: Instrumentation and Concepts

## Overview
Advanced X-Ray concepts: how to instrument your code with the X-Ray SDK, the difference between annotations and metadata, and how sampling rules control costs.

## Key Concepts

### Instrumentation
- **Instrumentation** = measuring performance, diagnosing errors, writing trace information
- To instrument your app: import the **X-Ray SDK** and add minimal code (sometimes just config changes)
- Example (Node.js): require the X-Ray SDK and use it in your Express app
- You can customize with **interceptors, filters, handlers, and middleware**

### X-Ray Terminology Recap

| Term | Definition |
|------|-----------|
| **Segment** | Data sent by each application/service |
| **Sub-segment** | More granular detail within a segment |
| **Trace** | All segments collected together → end-to-end view of a request |
| **Sampling** | Controls how many requests are sent to X-Ray (to reduce cost) |
| **Annotations** | Key-value pairs, **indexed**, used to **search and filter** traces |
| **Metadata** | Key-value pairs, **NOT indexed**, cannot be used for searching |

### Annotations vs Metadata (EXAM IMPORTANT)
- **Annotations** = indexed → you CAN search/filter traces by them
- **Metadata** = NOT indexed → you CANNOT search by them, just extra info attached to traces

### Cross-Account Tracing
- X-Ray daemon can send traces to a **different account**
- Requires correct **IAM permissions** (daemon assumes the correct role)
- Use case: central account for all application tracing

### Sampling Rules

**Default rule:**
- **Reservoir:** 1 request per second (maximum requests to sample per second)
- **Rate:** 5% of additional requests beyond the reservoir

```
First request each second → ALWAYS recorded (reservoir = 1)
Remaining requests → 5% sampled (rate = 0.05)
```

**Custom sampling rules examples:**

| Rule | Reservoir | Rate | Effect |
|------|-----------|------|--------|
| Higher rate (e.g., POST requests) | 10 | 10% | Up to 10 req/s sampled + 10% of the rest |
| Full debugging | 1 | 100% | ALL requests recorded (expensive!) |

**Key points:**
- Modify sampling rules **without changing code** — done in X-Ray console
- **No restart needed** — the X-Ray daemon automatically picks up new rules
- Full sampling (rate=1) is useful for temporary debugging but expensive in production

## Exam Tips
- **Annotations** = indexed, searchable. **Metadata** = not indexed, not searchable
- Default sampling: **1 req/s reservoir + 5% rate**
- Sampling rules changed in console → **no code change, no restart needed**
- Cross-account tracing → daemon assumes IAM role in central account
