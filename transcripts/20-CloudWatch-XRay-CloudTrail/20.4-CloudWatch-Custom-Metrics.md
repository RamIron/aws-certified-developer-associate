# 20.4 - CloudWatch Custom Metrics

> Udemy Lecture #238 · Section 20: AWS Monitoring & Audit: CloudWatch, X-Ray and CloudTrail

## Key Concepts

### 1. PutMetricData API

To push your own metrics to CloudWatch, use the **`PutMetricData`** API call. Use cases:
- RAM / memory usage
- Disk space
- Number of logged-in users
- Any application-specific metric

You define your own **namespace** (e.g., `MyNameSpace`) and **dimensions** (e.g., `InstanceId`, `InstanceType`, `Environment`) — whatever you want.

### 2. Metric Resolution

Two values for the **StorageResolution** API parameter:

| StorageResolution | Stores at | Read periods available | Retention at high-res |
|-------------------|-----------|----------------------|----------------------|
| **60** (standard) | 1 minute | 60s and multiples | Normal |
| **1** (high-res) | 1 second | **1s, 5s, 10s, 30s**, 60s+ | **3 hours** (then aggregated to 1 min) |

- `StorageResolution` is set **per API call**, not per metric
- The 1/5/10/30s intervals are **read periods** (how you query), not push intervals
- You push as often as you want; CloudWatch stores at the chosen resolution
- To "switch back" to standard, just send new data with `StorageResolution=60` — no toggle needed

### 3. Accepts Past & Future Timestamps

Custom metrics accept timestamps:
- Up to **2 weeks in the past**
- Up to **2 hours in the future**

No error is thrown — CloudWatch just accepts it. This means you must ensure your **EC2 instance time is correctly configured**, otherwise metrics will be out of sync.

### 4. Hands-On: Pushing via CLI

```
aws cloudwatch put-metric-data \
  --namespace MyNameSpace \
  --metric-name Buffers \
  --unit Bytes \
  --value 231434333 \
  --dimensions InstanceId=i-xxx,InstanceType=m5.large
```

- Creates a custom namespace visible in CloudWatch console
- The **CloudWatch Unified Agent** uses this same `PutMetricData` API internally to push metrics regularly

---

## Exam Tips

- Custom metric API = **`PutMetricData`**
- `StorageResolution=60` (standard) or `StorageResolution=1` (high-res, stored at 1s, readable at 1/5/10/30s, retained only 3 hours)
- Accepts timestamps **2 weeks past**, **2 hours future** — make sure EC2 time is correct
- CloudWatch Agent uses `PutMetricData` under the hood
