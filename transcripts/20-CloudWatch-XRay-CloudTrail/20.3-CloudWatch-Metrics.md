# 20.3 - CloudWatch Metrics

> Udemy Lecture #237 · Section 20: AWS Monitoring & Audit: CloudWatch, X-Ray and CloudTrail

## Key Concepts

### 1. What Are CloudWatch Metrics?

Every AWS service pushes metrics to CloudWatch. The metric name tells you what it measures (e.g., `CPUUtilization`, `NetworkIn`).

- **Namespace**: groups metrics by service (e.g., `AWS/EC2`, `AWS/ELB`)
- **Dimension**: attribute of a metric (e.g., `InstanceId`, `Environment`) — up to **30 dimensions** per metric
- **Timestamp**: every data point has one
- You can build **CloudWatch Dashboards** from metrics

### 2. EC2 Metrics & Detailed Monitoring

| Mode | Interval | Cost |
|------|----------|------|
| Default | Every **5 minutes** | Free |
| Detailed monitoring | Every **1 minute** | Additional cost |

Benefits of detailed monitoring:
- React faster to changing metrics
- ASG can scale in/out faster
- Gives you **10 detailed monitoring metrics**

### 3. RAM Is NOT a Default Metric

EC2 **memory usage (RAM) is not pushed by default**. You must push it yourself as a **custom metric** (covered in next lesson).

### 4. CloudWatch Console

- Left sidebar → Metrics → see all namespaces by service
- Filter by: region, dimension, resource
- View as: line, stacked area, number, pie chart
- Actions: add to dashboard, download CSV, share

## Exam Tips

- Default EC2 metrics = every **5 minutes**, detailed = every **1 minute**
- **RAM/memory is NOT a built-in metric** — must be pushed as custom metric
- Up to **30 dimensions** per metric
