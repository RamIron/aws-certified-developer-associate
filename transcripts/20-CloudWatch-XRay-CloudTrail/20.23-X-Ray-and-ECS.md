# 20.23 - X-Ray & ECS

## Overview
Three patterns for running X-Ray with ECS, plus a task definition example. This is a common exam topic.

## Three Patterns

### Pattern 1: Daemon Container (EC2 Launch Type only)
- One X-Ray daemon container runs on **every EC2 instance** in the cluster
- All app containers on that instance share the same daemon
- 10 EC2 instances = 10 X-Ray daemon containers

```
EC2 Instance A                EC2 Instance B
┌───────────────────┐        ┌───────────────────┐
│ App 1   App 2     │        │ App 3   App 4     │
│   │       │       │        │   │       │       │
│   └──┬────┘       │        │   └──┬────┘       │
│      ▼            │        │      ▼            │
│ X-Ray Daemon      │        │ X-Ray Daemon      │
└───────────────────┘        └───────────────────┘
```

### Pattern 2: Side Car (EC2 Launch Type)
- One X-Ray daemon container runs **alongside each app container**
- 20 app containers = 20 X-Ray side cars

```
EC2 Instance
┌──────────────────────────────┐
│ [App 1] + [X-Ray Sidecar]   │
│ [App 2] + [X-Ray Sidecar]   │
│ [App 3] + [X-Ray Sidecar]   │
└──────────────────────────────┘
```

### Pattern 3: Side Car (Fargate)
- Fargate = no control over EC2 instances → **cannot use Daemon pattern**
- Must use **Side Car** — one X-Ray container per Fargate task

```
Fargate Task 1               Fargate Task 2
┌─────────────────┐         ┌─────────────────┐
│ App + X-Ray     │         │ App + X-Ray     │
│ Sidecar         │         │ Sidecar         │
└─────────────────┘         └─────────────────┘
```

## Task Definition — Key Configuration (Side Car)

Three things to configure:

### 1. X-Ray Daemon container port
```json
{
  "name": "xray-daemon",
  "portMappings": [
    {
      "containerPort": 2000,
      "protocol": "udp"
    }
  ]
}
```

### 2. Environment variable on the app container
```json
{
  "name": "scorekeep-api",
  "environment": [
    {
      "name": "AWS_XRAY_DAEMON_ADDRESS",
      "value": "xray-daemon:2000"
    }
  ]
}
```

### 3. Link the containers (networking)
```json
{
  "links": ["xray-daemon"]
}
```

## Exam Tips
- **Fargate → Side Car only** (no Daemon option — you don't control the instances)
- **EC2 Launch Type → Daemon or Side Car** (both work)
- Remember: port **2000**, protocol **UDP**
- Environment variable: `AWS_XRAY_DAEMON_ADDRESS` = `xray-daemon:2000`
- Containers must be **linked** for networking
