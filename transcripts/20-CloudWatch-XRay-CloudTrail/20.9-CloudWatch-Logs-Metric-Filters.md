# 20.9 - CloudWatch Logs - Metric Filters

> Udemy Lecture #243 · Section 20: AWS Monitoring & Audit: CloudWatch, X-Ray and CloudTrail

## Key Concept

### Metric Filters: Turn Log Patterns into Metrics → Alarms

A metric filter scans your logs for a pattern (like a specific IP or the word "error") and creates a **CloudWatch Metric** from it. Then you can attach a **CloudWatch Alarm** to that metric.

```
EC2 → CW Agent → CloudWatch Logs → Metric Filter → CloudWatch Metric → Alarm → SNS
```

Example: "If the word `error` appears 5+ times in less than 1 minute → trigger an SNS notification"

### Important Details

- **NOT retroactive** — only filters events that happen **after** the filter is created (old logs are ignored)
- Up to **3 dimensions** per metric filter
- Filter expressions can match: specific IPs, error counts, keywords, patterns, etc.

---

## Exam Tips

- Metric filters are **not retroactive** — only new log events are counted
