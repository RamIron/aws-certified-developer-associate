# 20.25 - CloudTrail

## Overview
CloudTrail provides governance, compliance, and audit for your AWS accounts. It records a history of all events and API calls made by the console, SDK, CLI, IAM users/roles, and other AWS services. **Enabled by default.**

## Key Concepts

### What CloudTrail Does
- Records **who did what and when** in your AWS account
- Enabled by **default**
- Logs can be sent to **CloudWatch Logs** or **S3**
- A trail can cover **all regions** or a **single region**
- "Someone deleted an EC2 instance — who was it?" → **Look at CloudTrail**

### Three Types of Events

#### 1. Management Events (logged by default)
- Operations performed on resources in your AWS account
- Examples: `IAM:AttachRolePolicy`, create a subnet, set up logging
- **Logged by default** — trails always capture these
- Can separate into:
  - **Read Events** — don't modify resources (e.g., list IAM users, list EC2 instances)
  - **Write Events** — may modify resources (e.g., delete a DynamoDB table) — more destructive

#### 2. Data Events (NOT logged by default)
- High-volume operations on data — disabled by default to avoid huge costs
- **S3 object-level activity:** `GetObject` (read), `DeleteObject` / `PutObject` (write)
- **Lambda function execution:** `Invoke` API calls — how many times functions are invoked
- Can separate into Read and Write Events

#### 3. CloudTrail Insights Events (must be enabled, paid)
- Detects **unusual activity** in your account by analyzing management events
- CloudTrail creates a **baseline** of normal activity, then flags anomalies
- Examples: inaccurate resource provisioning, hitting service limits, bursts of IAM actions, gaps in maintenance
- Anomalies appear in: **CloudTrail console**, **Amazon S3**, and **EventBridge** (for automation)

```
Management Events (continuous)
        │
        ▼
CloudTrail Insights (analyzes baseline vs current)
        │
        ▼ (anomaly detected)
   ┌────┼────────────┐
   │    │             │
   ▼    ▼             ▼
Console  S3      EventBridge → email/automation
```

### Event Retention
- Default retention: **90 days** — then deleted
- For long-term storage: send to **S3**
- To analyze historical events: use **Athena** (serverless query service for S3)

```
All Events → CloudTrail (90 days) → S3 (long-term) → Athena (query/analyze)
```

## Exam Tips
- CloudTrail = **who did what and when** (audit/governance)
- Enabled by **default**
- Management Events = logged by default. Data Events = NOT logged by default
- Insights = detect unusual activity, must be enabled, costs money
- Beyond 90 days → **S3 + Athena**
- "Someone deleted X, who was it?" → **CloudTrail**
