# 20.22 - X-Ray with Beanstalk

## Overview
How to enable X-Ray on Elastic Beanstalk. The daemon is already included in Beanstalk platforms — you just need to turn it on.

## Key Concepts

### Enabling X-Ray on Beanstalk (2 ways)
1. **Console option** — In Beanstalk configuration → Monitoring → enable "Amazon X-Ray"
2. **`.ebextensions` file** — Create `.ebextensions/xray-daemon.config`:
```yaml
option_settings:
  aws:elasticbeanstalk:xray:
    XRayEnabled: true
```

### Still Required
- EC2 instance profile must have **X-Ray IAM permissions** (the default `aws-elasticbeanstalk-ec2-role` already includes them via `AWSElasticBeanstalkWebTier` policy)
- Application code must be **instrumented with X-Ray SDK**
- If using a **custom IAM role** (not the default Beanstalk role), you must add X-Ray permissions manually

### Multi-Docker Container Exception
- Beanstalk's built-in X-Ray daemon does **NOT** work with multi-Docker containers
- You need to manage the X-Ray daemon yourself (covered in next lesson with ECS)

## Exam Tips
- Beanstalk **includes the X-Ray daemon** — just enable it (console or `.ebextensions`)
- Default Beanstalk EC2 role already has X-Ray permissions
- Custom role → must add X-Ray permissions manually
- Multi-Docker → manage X-Ray daemon yourself
- You still need the **X-Ray SDK in your code** — enabling the daemon alone isn't enough
