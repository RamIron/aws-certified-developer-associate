# 20.27 - CloudTrail - EventBridge Integration

## Overview
CloudTrail logs every API call. EventBridge can react to those events. Together, you can create alerts for ANY API call in your AWS account.

## The Pattern

```
Any AWS API Call → CloudTrail (logs it) → EventBridge (reacts to it) → SNS/Lambda/etc.
```

## Examples

### Example 1: DynamoDB table deletion
```
User calls DeleteTable API
        │
        ▼
CloudTrail logs it → EventBridge rule matches "DeleteTable"
        │
        ▼
SNS → email: "Someone just deleted a DynamoDB table!"
```

### Example 2: IAM role assumption
```
User calls AssumeRole API (IAM)
        │
        ▼
CloudTrail → EventBridge → SNS notification
```

### Example 3: Security Group changes
```
User calls AuthorizeSecurityGroupIngress API (EC2)
        │
        ▼
CloudTrail → EventBridge → SNS notification
```

## Key Takeaway
- **Any API call** in AWS is logged by CloudTrail
- **All CloudTrail events** appear in EventBridge
- You can create an EventBridge **rule** to match any specific API call
- The possibilities are endless — monitor anything that happens in your account

## Exam Tips
- CloudTrail + EventBridge = **react to any API call** in real time
- Common pattern: CloudTrail → EventBridge → SNS (alert) or Lambda (automate)
