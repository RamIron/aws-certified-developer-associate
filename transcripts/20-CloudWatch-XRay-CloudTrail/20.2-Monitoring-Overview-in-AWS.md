# 20.2 - Monitoring Overview in AWS

> Udemy Lecture #236 · Section 20: AWS Monitoring & Audit: CloudWatch, X-Ray and CloudTrail

## Why Monitoring Matters

You know how to deploy — Beanstalk, IaC, highly available architectures. But **users don't care how you deployed**. They care that:

- **Latency** stays low (and doesn't silently increase over time)
- **Outages** don't degrade their experience
- **They never have to report the problem** — you should detect it first

Internally, monitoring lets you:
- Prevent issues before they happen
- Detect problems before users do
- Track performance and cost trends
- Learn patterns and improve

## The Three Pillars of AWS Monitoring

```
┌─────────────────────────────────────────────────────┐
│              AWS Monitoring Stack                     │
├──────────────┬──────────────┬────────────────────────┤
│  CloudWatch  │    X-Ray     │     CloudTrail         │
├──────────────┼──────────────┼────────────────────────┤
│ • Metrics    │ • App perf   │ • API call audit       │
│ • Logs       │ • Errors     │ • Who changed what     │
│ • Events     │ • Distributed│ • Resource change      │
│ • Alarms     │   tracing    │   tracking             │
├──────────────┼──────────────┼────────────────────────┤
│ What's       │ Why is it    │ Who did                │
│ happening?   │ slow/broken? │ what?                  │
└──────────────┴──────────────┴────────────────────────┘
```

### CloudWatch — Observability
- **Metrics**: collect and track numerical data
- **Logs**: monitor and analyze log files
- **Events**: send notifications when things happen
- **Alarms**: react in real-time to metrics, events, and logs

### X-Ray — Troubleshooting
- Troubleshoot **application performance and errors**
- **Distributed tracing** of microservices (trace calls across services: S3, DynamoDB, Lambda, etc.)
- See how long each call takes, end-to-end

### CloudTrail — Audit
- Internal monitoring of **API calls** being made
- Audit **changes** to AWS resources by users
- Who did what, and when

## Exam Tips

- "What's happening in my app?" → **CloudWatch**
- "Why is my app slow?" → **X-Ray**
- "Who changed this resource?" → **CloudTrail**
