# 20.12 - CloudWatch Alarms Hands On

## Overview
Hands-on demo creating a CloudWatch Alarm that terminates an EC2 instance when CPU utilization stays above 95% for 15 minutes, plus testing with `set-alarm-state`.

## Key Takeaways

### Alarm Setup Steps
1. Launch a t2.micro EC2 instance
2. Create alarm → Select metric → EC2 Per-Instance Metrics → CPU Utilization
3. Configure threshold: **Greater than 95%** for **3 out of 3** data points (= 15 minutes at 5-min intervals)
4. Choose action: **EC2 Action → Terminate instance** (when in ALARM state)

### Alarm Behavior
- New alarms start in **INSUFFICIENT_DATA** state (need ~15 min of data to evaluate)
- Transitions to **OK** once enough data points are collected
- Transitions to **ALARM** only when threshold is breached for the configured data points

### Testing with `set-alarm-state`
```bash
aws cloudwatch set-alarm-state \
  --alarm-name "terminate-EC2-on-high-CPU" \
  --state-value ALARM \
  --state-reason "testing"
```

- Forces the alarm into ALARM state immediately
- The terminate action executed successfully
- EC2 instance transitioned to **shutting-down → terminated**
- Alarm history shows: OK → ALARM → Action executed

### Evaluation Period Options
- **3 out of 3**: all 3 consecutive data points must breach (strict)
- Configurable: e.g., 2 out of 3, 1 out of 5, etc.

## Exam Tips
- `set-alarm-state` is the way to **test alarms without generating real load**
- EC2 actions include: Stop, Terminate, Reboot, Recover
- Default metric period for EC2 without detailed monitoring = **5 minutes**
