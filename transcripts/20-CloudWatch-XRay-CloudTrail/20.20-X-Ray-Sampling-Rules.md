# 20.20 - X-Ray: Sampling Rules

## Overview
Hands-on demo showing how to configure sampling rules in the CloudWatch/X-Ray console.

## Key Takeaways

### Where to Find Sampling Rules
- CloudWatch console → left sidebar → **Settings** → **Traces** section
- Under settings: Encryption rules, Group rules, **Sampling rules**

### Default Sampling Rule
- Priority: **10,000** (lowest priority)
- Reservoir: **1 request per second**
- Fixed rate: **5%**
- Matching criteria: **everything** (cannot be changed — it's the catch-all default)
- You CAN edit the reservoir size and fixed rate, but NOT the matching criteria

### Creating Custom Sampling Rules
- Priority: **1 to 9,999** (lower number = higher priority)
- A rule with priority 5,000 takes precedence over the default (10,000)
- Configurable fields:
  - **Reservoir size** — max requests to sample per second
  - **Fixed rate** — percentage of additional requests
  - **Service name** — target a specific service (e.g., "MYSERVICE")
  - **HTTP method** — filter by method (e.g., POST only)
  - **URL path** — filter by specific endpoint path

### Example Custom Rule
- Name: DemoSampling
- Priority: 5,000
- Reservoir: 1
- Rate: 100%
- Service: MYSERVICE
- Method: POST
- Effect: trace ALL POST requests to MYSERVICE

### No Restart Needed
- Once you create/edit a sampling rule in the console, X-Ray daemons **automatically** pick up the changes
- No code changes, no daemon restart required

## Exam Tips
- Default rule priority = **10,000** (always evaluated last)
- Custom rules: **lower priority number = evaluated first**
- Custom rules can target specific services, methods, and URL paths
- Changes take effect **automatically** — no restart needed
