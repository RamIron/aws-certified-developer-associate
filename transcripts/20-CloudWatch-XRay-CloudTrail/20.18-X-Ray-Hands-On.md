# 20.18 - X-Ray Hands On

## Overview
Hands-on demo deploying a sample Tic Tac Toe app (Scorekeep) on ECS via CloudFormation, then exploring X-Ray's Service Map and Traces console.

## Key Takeaways

### X-Ray Console Location
- X-Ray is now **inside the CloudWatch console** (left sidebar → Service Map)
- Makes sense: metrics, logs, alarms, and traces all in one place
- Some features (sampling, encryption, groups) still only in the old X-Ray console

### Demo Setup
- Deployed via **CloudFormation** template (ECS cluster with frontend + backend)
- App uses DynamoDB tables, SNS topics — all instrumented with X-Ray
- Playing a Tic Tac Toe game generates traces

### Service Map
- Shows **visual dependencies** between all components
- ECS container → multiple DynamoDB tables → SNS topic
- Errors highlighted in **orange** (e.g., SNS returning 100% errors)
- Click a node to see: average latency, request count, faults, response time distribution

### Traces Console
- Query and filter traces by service/node
- Each trace shows:
  - **Trace map** — simplified service map for that one request
  - **Segment timeline** — breakdown of each API call with latency
  - **Segment details** — errors, resources, annotations, metadata
- Example: one request had 500ms latency → drill down → see which DynamoDB calls caused it

### What You Can See Per Trace
- The specific API calls made (e.g., GetItem on DynamoDB)
- Latency of each individual call
- Errors at each step
- Annotations and metadata
- Corresponding CloudWatch Logs (if enabled)

## Exam Tips
- X-Ray service map = **visual graph of all service dependencies**
- Traces let you drill down into **individual requests** to find latency/errors
- X-Ray + CloudWatch Logs = see traces AND corresponding logs together
- Remember to **delete CloudFormation stacks** after hands-on labs
