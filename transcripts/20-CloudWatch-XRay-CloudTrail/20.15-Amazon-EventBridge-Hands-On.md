# 20.15 - Amazon EventBridge - Hands On

## Overview
Hands-on walkthrough of EventBridge: creating event pattern rules, schedules, custom event buses, archives, replay, partner event sources, API destinations, and the Schema Registry.

## Key Takeaways

### EventBridge Console Options
1. **Rules** (event pattern) — react to events
2. **Schedules** — cron/rate-based triggers (newer feature, separate from rules)
3. **Pipes** — send data from source to target with optional filtering/enrichment
4. **Schema Registry** — view schemas of events

### Demo 1: Event Pattern Rule
- **Goal:** Get notified when an EC2 instance is shut down or terminated
- **Event source:** EC2 Instance State-change Notification
- **Filter:** `state` equals `shutting-down` OR `terminated`
- **Target:** SNS Topic → email notification
- An IAM role is automatically created for EventBridge to publish to SNS
- Retry policy and DLQ options available
- Rules can be **disabled** when not needed

### Sample Event Schema
```json
{
  "source": "aws.ec2",
  "detail-type": "EC2 Instance State-change Notification",
  "detail": {
    "instance-id": "i-1234567890abcdef0",
    "state": "shutting-down"
  }
}
```

### Demo 2: EventBridge Schedule
- **Goal:** Invoke a Lambda function every hour
- Schedule types: **One-time**, **Cron-based**, or **Rate-based**
- Example: Rate = 1 hour, no flexible time window
- Target options: ECS task, Kinesis Data Firehose, Lambda function, etc.

### Other Features Shown
- **Event Buses:** Default (AWS events) + can create Custom event buses for your own apps
- **Archives:** Archive events from a bus for later inspection
- **Replay:** Replay past archived events
- **Partner Event Sources:** Third-party SaaS (Auth0, etc.) sending events directly into your account
- **API Destinations:** Connect EventBridge to external HTTP endpoints (your own infrastructure)
- **Schemas:** Browse all AWS event schemas or create custom registries

## Exam Tips
- EventBridge **Schedules** are now a separate feature (not just rules with cron)
- API Destinations = EventBridge → external HTTP endpoint
- Partner Event Sources = third-party SaaS → EventBridge
