# 20.24 - AWS Distro for OpenTelemetry

## Overview
AWS-supported distribution of the open-source OpenTelemetry project for collecting traces, metrics, and metadata from your applications.

## Why It Exists
- X-Ray is AWS-specific. OpenTelemetry is **open-source and vendor-neutral**
- Lets you send traces/metrics to **multiple destinations** simultaneously (X-Ray + Datadog, for example)
- Standardized APIs across your organization

## What It Does
- Collects **distributed traces** and **metrics** from your applications
- Collects **metadata** from AWS resources and services
- Agents can **auto-instrument** your code (collect traces without code changes)
- Sends data to multiple destinations at once

## Destinations

```
Your Apps (EC2, ECS, EKS, Fargate, Lambda, on-premises)
        │
        ▼
AWS Distro for OpenTelemetry (collector)
        │
        ├──→ AWS X-Ray (traces)
        ├──→ Amazon CloudWatch (metrics)
        ├──→ Amazon Managed Prometheus (traces + metrics)
        └──→ Partner solutions (Datadog, etc.)
```

## OpenTelemetry vs X-Ray

| | X-Ray | AWS Distro for OpenTelemetry |
|---|---|---|
| **Type** | AWS proprietary | Open-source (AWS-supported distro) |
| **Code changes** | Need X-Ray SDK | Can auto-instrument (no code changes) |
| **Destinations** | X-Ray only | Multiple destinations simultaneously |
| **Migration reason** | — | Standardize on open-source APIs |

## When to Migrate from X-Ray to OpenTelemetry
1. You want to **standardize with open-source APIs** across your org
2. You need to send traces to **multiple destinations simultaneously**

## Exam Tips
- Usually **one high-level question** at most
- OpenTelemetry = **open-source**, multi-destination, auto-instrumentation
- X-Ray = AWS-specific, single destination
- If question mentions "open-source tracing" or "send traces to multiple destinations" → **AWS Distro for OpenTelemetry**
