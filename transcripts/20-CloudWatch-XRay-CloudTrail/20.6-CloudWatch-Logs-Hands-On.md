# 20.6 - CloudWatch Logs - Hands On

> Udemy Lecture #240 · Section 20: AWS Monitoring & Audit: CloudWatch, X-Ray and CloudTrail

## Hands-On Summary

### Log Groups & Streams in the Console

- Console shows all log groups (created by Lambda, DataSync, Glue, SSM RunCommand, etc.)
- Inside a log group: multiple **log streams** (one per instance, with `stdout` and `stderr` variants)
- You can search within logs using keywords (e.g., `http`, `installing`)

### Metric Filters (from the Console)

- Create a **metric filter** on a log group with a filter pattern (e.g., `"installing"`)
- Test the pattern against log streams to see matches
- Assign it a metric name + namespace (e.g., `DemoMetricFilter` / `DemoMetric`)
- Set a **metric value** (e.g., `1`) — counts how many times the pattern matches
- You can then create a **CloudWatch Alarm** on top of this metric filter

### Subscription Filters

- Available destinations from console: **OpenSearch, Kinesis Data Streams, Kinesis Data Firehose, Lambda**
- Limit: up to **2 subscription filters** per log group

### Retention Settings

- Editable per log group: **never expire** up to **120 months (10 years)**

### Export to S3

- Choose date range, stream prefix, S3 bucket + prefix
- This is the batch export (up to 12 hours, `CreateExportTask`)

### Creating a Log Group

- Specify: name, retention settings, optional **KMS key** for encryption

### CloudWatch Logs Insights Demo

- Query language to search log groups
- Select time range (e.g., past 60 days) and run query
- Save queries for reuse
- Built-in sample queries: Lambda latency stats, VPC Flow Logs top transfers, etc.

---

## Extra Exam Detail from Hands-On

- **2 subscription filters max** per log group
- Metric filters let you turn **log patterns into CloudWatch metrics** → then create alarms on them
