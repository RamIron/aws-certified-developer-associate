# 20.8 - CloudWatch Agent & CloudWatch Logs Agent

> Udemy Lecture #242 · Section 20: AWS Monitoring & Audit: CloudWatch, X-Ray and CloudTrail

## Key Concepts

### 1. Why You Need an Agent

By default, **no logs go from EC2 to CloudWatch**. You must install an agent (small program) on the instance to push logs. The EC2 instance needs an **IAM role** that allows writing to CloudWatch Logs.

Also works on **on-premises servers** (Linux VMs, VMware, etc.) — same agent, same result.

> **Important distinction: Metrics ≠ Logs**
> - **Metrics** (numbers like CPU 75%) → EC2 pushes these by default (every 5 min)
> - **Logs** (text lines like error messages) → EC2 pushes **nothing** by default → need agent

### 2. Two Agents: Old vs New

| | CloudWatch Logs Agent (OLD) | CloudWatch Unified Agent (NEW) |
|--|----------------------------|-------------------------------|
| **Logs** | ✅ Sends to CW Logs | ✅ Sends to CW Logs |
| **Metrics** | ❌ No | ✅ System-level metrics (RAM, processes, swap, etc.) |
| **Config** | Manual | **SSM Parameter Store** (centralized config) |
| **Status** | Deprecated | **Use this one** |

### 3. Unified Agent Metrics (Granular)

Way more detailed than default EC2 metrics:

| Category | What you get |
|----------|-------------|
| **CPU** | active, guest, idle, system, user, steal |
| **Disk** | free, used, total |
| **Disk I/O** | writes, reads, bytes, IOPS |
| **RAM** | free, inactive, used, total, cached |
| **Netstat** | TCP/UDP connections, net packets, bytes |
| **Processes** | total, dead, blocked, idle, running, sleeping |
| **Swap** | free, used, used % |

### 4. Default EC2 vs Unified Agent

| | Default EC2 Monitoring | Unified Agent |
|--|----------------------|---------------|
| **CPU** | ✅ High-level only | ✅ Granular (active, idle, steal, etc.) |
| **Disk** | ✅ High-level | ✅ Granular + I/O |
| **Network** | ✅ High-level | ✅ Granular (TCP/UDP, packets) |
| **RAM/Memory** | ❌ Not available | ✅ Yes |
| **Swap** | ❌ Not available | ✅ Yes |
| **Processes** | ❌ Not available | ✅ Yes |

## Exam Tips

- No logs from EC2 by default → need **CloudWatch agent** + **IAM role**
- **Unified Agent** = logs + metrics (use this one). Logs Agent = logs only (deprecated)
- Unified Agent configured via **SSM Parameter Store** (centralized)
- "More granular metrics" or "RAM/memory/swap/processes" → **CloudWatch Unified Agent**
- Works on **EC2 and on-premises** servers

## Tricky Exam Questions

**Q: How to monitor EC2 instance memory usage in CloudWatch?**
- A: Use **Unified CloudWatch Agent** to push memory as a custom metric
- Trap: "Enable Detailed Monitoring" — this only changes interval from 5min to 1min on the SAME default metrics (CPU, disk, network). Still no RAM.
- Trap: "EC2 pushes memory by default" — **false**. RAM is never a default metric.
