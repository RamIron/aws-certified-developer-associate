# 10.6 - Three Tier Architecture

## Key Concepts

### 1. Three Tier Solution Architecture

```
┌─────────────────────────────────────────────────────┐
│                        VPC                          │
│                                                     │
│  Tier 1: PUBLIC SUBNET (Web Tier)                   │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐            │
│  │   ELB   │  │   ELB   │  │   ELB   │            │
│  │  AZ 1   │  │  AZ 2   │  │  AZ 3   │            │
│  └────┬────┘  └────┬────┘  └────┬────┘            │
│       │             │            │                  │
│  Tier 2: PRIVATE SUBNET (Application Tier)          │
│  ┌────┴────┐  ┌────┴────┐  ┌────┴────┐            │
│  │   EC2   │  │   EC2   │  │   EC2   │  ← ASG     │
│  │  AZ 1   │  │  AZ 2   │  │  AZ 3   │            │
│  └────┬────┘  └────┬────┘  └────┬────┘            │
│       │             │            │                  │
│  Tier 3: PRIVATE SUBNET (Data Tier)                 │
│  ┌─────────────────────────────────────┐           │
│  │  Amazon RDS    │   ElastiCache      │           │
│  └─────────────────────────────────────┘           │
└─────────────────────────────────────────────────────┘

Route 53 ──► ELB (public) ──► EC2 (private) ──► RDS + ElastiCache (private)
```

- **Tier 1 (Web):** ELB in **public subnets** — publicly accessible
- **Tier 2 (App):** EC2 in ASG in **private subnets** — only accessible from ELB
- **Tier 3 (Data):** RDS + ElastiCache in **private subnets** — only accessible from EC2

### 2. LAMP Stack on EC2

| Letter | Component | AWS Service |
|--------|-----------|-------------|
| **L** | Linux | EC2 OS |
| **A** | Apache | Web server on EC2 |
| **M** | MySQL | RDS |
| **P** | PHP | Application logic on EC2 |

- Add **ElastiCache** (Redis/Memcached) for caching
- Add **EBS** for local storage on EC2

### 3. Wordpress Architecture

- ELB + EC2 instances in ASG
- **EFS** used to share images across all EC2 instances (shared network drive)
- EFS creates ENIs in each AZ so all instances can access the same files

---

## Exam Tips
- Three tier = ELB (public) → EC2/ASG (private) → RDS/ElastiCache (private)
- ELB must be in **public subnet**, compute and data in **private subnets**
- ElastiCache: caching data from RDS + storing session data
- EFS: shared storage across multiple EC2 instances in different AZs
- LAMP = Linux, Apache, MySQL, PHP
