# 10.3 - NACL, SG, VPC Flow Logs

## Key Concepts

### 1. Two Layers of Network Security

Traffic from the internet passes through **two firewalls** before reaching your EC2 instance:

```
Internet ──► NACL (subnet level) ──► Security Group (instance level) ──► EC2
```

### 2. NACL vs Security Group

| | NACL (Network ACL) | Security Group |
|--|-------------------|----------------|
| **Level** | Subnet | Instance / ENI |
| **Rules** | Allow AND Deny | Allow only |
| **Stateful?** | **Stateless** (must allow both inbound AND outbound) | **Stateful** (return traffic auto-allowed) |
| **References** | IP addresses only | IP addresses or other Security Groups |
| **Default** | Allows everything in/out | Denies all inbound, allows all outbound |

### 3. VPC Flow Logs

- Captures **all IP traffic** going through your VPC interfaces
- Three levels: **VPC** flow logs, **Subnet** flow logs, **ENI** flow logs
- Logs both **allowed and denied** traffic
- Captures traffic from AWS managed services too (ELB, ElastiCache, RDS, Aurora)
- Use case: **troubleshoot connectivity issues** (why can't subnet A talk to subnet B?)

Flow logs can be sent to:
- **Amazon S3**
- **CloudWatch Logs**
- **Kinesis Data Firehose**

---

## Exam Tips
- NACL = subnet level, allow + deny, stateless, IP addresses only
- Security Group = instance level, allow only, stateful
- Default NACL allows everything (that's why we never changed it in the course)
- VPC Flow Logs = troubleshoot network connectivity issues
- Flow logs capture traffic from AWS managed services (ELB, RDS, etc.)
- Flow logs destinations: S3, CloudWatch Logs, Kinesis Data Firehose
