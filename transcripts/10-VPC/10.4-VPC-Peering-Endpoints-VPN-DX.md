# 10.4 - VPC Peering, Endpoints, VPN, DX

## Key Concepts

### 1. VPC Peering

- Connects two VPCs **privately** using AWS network (as if same network)
- Works across **different accounts** and **different regions**
- IP ranges must **NOT overlap**
- **Not transitive** — must create a peering connection for each pair

```
VPC A ◄──peering──► VPC B
VPC A ◄──peering──► VPC C
VPC B ◄──── ✕ ────► VPC C   (no transitivity, needs its own peering)
```

### 2. VPC Endpoints (important for exam)

- Connect to AWS services using **private network** instead of public internet
- Provides better **security** and **lower latency**

| Type | Used for | How it works |
|------|----------|-------------|
| **Gateway Endpoint** | **S3 and DynamoDB** only | Gateway in VPC |
| **Interface Endpoint** | All other AWS services | ENI in your subnet |

```
[EC2 in Private Subnet] ──► VPC Endpoint Gateway ──► S3 / DynamoDB
                                (private, no internet)

[EC2 in Private Subnet] ──► VPC Endpoint Interface (ENI) ──► CloudWatch / etc.
                                (private, no internet)
```

### 3. Site-to-Site VPN

- Connects **on-premises data center** to VPC
- Encrypted, goes over the **public internet**
- Easy and quick to set up (minutes)

### 4. Direct Connect (DX)

- Connects **on-premises data center** to VPC
- **Physical private connection** (does NOT go over the internet)
- Secure, fast, low latency
- Takes **at least a month** to establish

| | Site-to-Site VPN | Direct Connect |
|--|-----------------|----------------|
| **Goes over** | Public internet (encrypted) | Private network |
| **Setup time** | Minutes | At least 1 month |
| **Connection** | Virtual | Physical |

---

## Exam Tips
- VPC Peering = connect 2 VPCs privately, NOT transitive, no overlapping CIDRs
- **VPC Endpoints** = private access to AWS services (exam favorite!)
  - Gateway = S3 + DynamoDB | Interface = everything else
- "Privately connect to AWS service from VPC" → **VPC Endpoint**
- Site-to-Site VPN = encrypted over public internet, quick setup
- Direct Connect = private physical line, at least 1 month
