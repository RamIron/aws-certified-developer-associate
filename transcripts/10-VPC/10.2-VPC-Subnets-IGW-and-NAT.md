# 10.2 - VPC, Subnets, IGW and NAT

## Key Concepts

### 1. VPC and Subnets

- **VPC (Virtual Private Cloud)** — a private network within AWS to deploy your resources
- VPC is a **regional resource** (one VPC per region)
- **Subnets** — partition your network inside a VPC, defined at the **AZ level**
- **Route Tables** — define how traffic flows between subnets and the internet

| Subnet Type | Internet Access |
|-------------|----------------|
| **Public** | Accessible from/to the internet |
| **Private** | NOT accessible from the internet |

```
┌─────────────── VPC (region) ──────────────────┐
│   CIDR: 10.0.0.0/16 (IP range)               │
│                                                │
│  ┌─── AZ 1 ───────┐   ┌─── AZ 2 ───────┐    │
│  │ Public Subnet   │   │ Public Subnet   │    │
│  │  [EC2]          │   │  [EC2]          │    │
│  │ Private Subnet  │   │ Private Subnet  │    │
│  │  [EC2] [RDS]    │   │  [EC2] [RDS]    │    │
│  └─────────────────┘   └─────────────────┘    │
└────────────────────────────────────────────────┘
```

- **Default VPC**: created automatically per region, has only **public subnets** (one per AZ)

### 2. Internet Gateway (IGW)

- Allows VPC instances in **public subnets** to connect to the internet
- Lives inside the VPC
- A public subnet = a subnet with a **route to the IGW**

```
[EC2 in Public Subnet] ──► Route Table ──► Internet Gateway ──► Internet
```

### 3. NAT Gateway / NAT Instance

- Allows instances in **private subnets** to access the internet (e.g., software updates) while **remaining private** (internet can't reach them)
- **NAT Gateway** = managed by AWS (no provisioning/scaling needed)
- **NAT Instance** = self-managed (you manage the EC2 instance)
- Deployed in a **public subnet**, private subnet routes through it

```
[EC2 in Private Subnet] ──► NAT Gateway (in public subnet) ──► IGW ──► Internet

Internet ──✕──► Cannot reach private subnet directly
```

---

## Exam Tips
- VPC = regional, Subnets = AZ level
- Public subnet = has route to **Internet Gateway**
- Private subnet needs a **NAT Gateway** (in public subnet) to access internet
- NAT Gateway = AWS managed, NAT Instance = self managed
- Default VPC has only public subnets
- NAT will come back when studying **Lambda functions**
