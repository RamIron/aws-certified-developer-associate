01 — EC2 Basics
Service Model
IaaS = Infrastructure as a Service — you rent virtual machines from AWS. You manage the OS and above.
User Data
Bootstrap script that runs ONCE at first launch only, executed as root. Used to install software, configure settings on boot.
EC2 Ecosystem
EC2 instances (compute)
EBS volumes (block storage)
ELB (load balancing)
Auto Scaling Groups
Instance Naming
m5.2xlarge

m = instance class
5 = generation
2xlarge = size within class
02 — Instance Type Categories
General Purpose
t, m families
Balanced compute, memory, and network. Best for web servers, code repos, small databases.
"Jack of all trades"
Compute Optimized
C family (c5, c6g...)
High-performance processors. Best for batch processing, media transcoding, HPC, ML inference, gaming servers.
"CPU go brrrr"
Memory Optimized
R, X1, z1 families
Large datasets in RAM. Best for in-memory databases (Redis), real-time processing of big unstructured data.
"R = RAM"
Storage Optimized
I, D, H1 families
High sequential read/write to large datasets on local storage. Best for OLTP, NoSQL, data warehousing, Hadoop.
"I = I/O, D = Disk, H = HDD"
03 — Security Groups
Only ALLOW rules — you cannot write explicit deny rules in a security group
Live OUTSIDE the EC2 instance — EC2 never sees blocked traffic; it is dropped at the SG level
Default behavior: all inbound traffic BLOCKED, all outbound traffic ALLOWED
Can reference other Security Groups as a source/destination — not just IP ranges or CIDR blocks
Stateful: return traffic is automatically allowed. If inbound is allowed, the response is allowed outbound
Troubleshooting Guide
🔴 Symptom: Timeout
Security Group is BLOCKING
Connection times out = traffic never reaches the instance. Check your inbound SG rules — the port or source IP is not allowed.
🟡 Symptom: Connection Refused
Application Problem
Traffic REACHED the instance. The SG is fine. The app is not running, crashed, or is listening on wrong port.
Classic Ports to Know
Port Protocol Description
22 TCP SSH — Secure Shell login into Linux instances
21 TCP FTP — File Transfer Protocol (upload files)
22 TCP SFTP — Secure FTP (also uses port 22, over SSH)
80 TCP HTTP — Unsecured web traffic
443 TCP HTTPS — Secured web traffic (TLS/SSL)
3389 TCP RDP — Remote Desktop Protocol (Windows instances)
04 — EC2 Purchasing Options
On-Demand
No Discount No Commitment
Best for: Short-term, unpredictable workloads. Pay by the second (Linux/Windows) or hour.
Reserved Instances
~72% Discount 1 or 3 Years
Best for: Steady-state workloads with predictable usage — databases, web servers with constant traffic.
Convertible Reserved
~66% Discount 1 or 3 Years
Best for: Steady-state but need flexibility to change instance type, OS, or tenancy during the term.
Savings Plans
~70% Discount 1 or 3 Years
Best for: Commit to a $/hour spend. Flexible on instance size, OS, and region. Excess billed on-demand.
Spot Instances
~90% Discount Can be interrupted
Best for: Batch jobs, data analysis, image processing, distributed resilient workloads. NEVER use for DBs.
Dedicated Hosts
Most Expensive On-Demand or 1–3y
Best for: Bring Your Own License (BYOL), compliance requirements. You get visibility into physical sockets and cores.
Dedicated Instances
No Discount No Commitment
Best for: Hardware dedicated to your account (no hardware sharing with other AWS customers) but no BYOL support.
Capacity Reservations
No Discount No Commitment
Best for: Reserve AZ capacity for any duration. You pay full on-demand rate whether or not instances are running.
Hotel Analogy On-Demand = walk in, full price  |  Reserved = book 1–3 years, big discount  |  Savings Plan = commit to $X/month, change room type freely  |  Spot = last-minute deal, can be kicked out if hotel needs the room  |  Dedicated Host = rent the whole building (bring your own license)  |  Capacity Reservation = reserve a room you might not use, still pay full price
05 — Exam Tips
When does User Data run?
Once at first boot only, executed as root
What service type is EC2?
IaaS — Infrastructure as a Service
Timeout connecting to EC2
Security group is blocking — check inbound rules
Connection refused on EC2
App problem — traffic reached the instance, check the application
SSH port / RDP port
22 (Linux SSH) · 3389 (Windows RDP)
Cheapest EC2 option
Spot (~90% off) — but can be interrupted at any time
NEVER use Spot for...
Databases or critical/stateful workloads — interruption risk
Dedicated Host vs Dedicated Instance
Host = whole physical server (BYOL/compliance). Instance = isolated hardware, shared within your account
Capacity Reservation discount?
None — it only guarantees AZ capacity availability
Savings Plans vs Reserved
Savings Plans: lock instance family + region, flexible size/OS. Reserved: lock specific type + region + OS
