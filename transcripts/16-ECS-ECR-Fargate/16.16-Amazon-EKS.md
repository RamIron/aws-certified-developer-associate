# 16.16 - Amazon EKS

## Key Concepts

### 1. What is EKS?

- **Elastic Kubernetes Service** — managed Kubernetes on AWS
- **Kubernetes** = open-source system for deploying, scaling, and managing containers
- Alternative to ECS — similar goal, different API

### 2. EKS vs ECS

| | ECS | EKS |
|---|-----|-----|
| **API** | AWS proprietary | Kubernetes (open-source) |
| **Portability** | AWS only | Cloud-agnostic (Azure, GCP, on-premises) |
| **Terminology** | Tasks, Services | Pods, Services |
| **Use when** | AWS-native, no Kubernetes experience | Already using Kubernetes, or need multi-cloud portability |

- ECS Task = EKS **Pod** (same idea, different name)

### 3. Launch Modes (same as ECS)

- **EC2** — you manage worker nodes (EC2 instances)
- **Fargate** — serverless, no nodes to manage

### 4. Node Types

| Type | Description |
|------|-------------|
| **Managed Node Groups** | AWS creates/manages EC2 nodes via ASG, supports On-Demand + Spot |
| **Self-Managed Nodes** | You create, register, and manage nodes yourself (more control) |
| **Fargate** | No nodes at all — serverless |

### 5. Data Volumes (Storage)

Uses **Container Storage Interface (CSI)** drivers:

| Storage | Works with Fargate? |
|---------|-------------------|
| Amazon EBS | No |
| Amazon EFS | **Yes** (only one that works with Fargate) |
| Amazon FSx for Lustre | No |
| Amazon FSx for NetApp ONTAP | No |

---

## Exam Tips
- **EKS** = managed Kubernetes, **ECS** = AWS-native containers
- "Already using Kubernetes" or "multi-cloud" → **EKS**
- ECS Task = EKS **Pod**
- EKS supports **EC2 and Fargate** (same as ECS)
- EKS + Fargate storage → only **EFS** works
- "Cloud-agnostic container orchestration" → **Kubernetes / EKS**
