# 16.5 - Amazon ECS - Auto Scaling

## Key Concepts

### 1. ECS Service Auto Scaling

Automatically increase/decrease the number of **ECS tasks** using **AWS Application Auto Scaling**.

Three metrics to scale on:

| Metric | What it measures |
|--------|-----------------|
| **CPU Utilization** | Average CPU usage of the ECS service |
| **Memory Utilization** | Average RAM usage of the ECS service |
| **ALB Request Count Per Target** | Number of requests per target from the ALB |

### 2. Auto Scaling Types

| Type | How it works |
|------|-------------|
| **Target Tracking** | Set a target value (e.g., "keep CPU at 70%") — scales to maintain it |
| **Step Scaling** | Scale based on CloudWatch Alarm thresholds |
| **Scheduled Scaling** | Scale ahead of time for predictable changes |

### 3. Task Scaling ≠ EC2 Scaling

- Scaling the **ECS Service** = adding/removing **tasks**
- Scaling the **EC2 cluster** = adding/removing **EC2 instances** (only for EC2 launch type)
- These are **two separate things** — more tasks need more capacity underneath
- With **Fargate**: only task scaling matters (no EC2 to manage)

### 4. EC2 Launch Type: How to Scale the Cluster

| Method | How it works |
|--------|-------------|
| **ASG Scaling** | Scale ASG based on CPU utilization (basic) |
| **ECS Cluster Capacity Provider** (recommended) | Automatically adds EC2 instances when there's not enough CPU/RAM to launch new tasks |

- Capacity Provider is paired with an ASG
- Smarter: it knows exactly when more capacity is needed
- **Exam: always prefer Capacity Provider** over plain ASG scaling

### 5. Full Auto Scaling Flow

```
More users → CPU usage increases
  → CloudWatch Metric (ECS service CPU)
    → CloudWatch Alarm triggers
      → Application Auto Scaling increases desired tasks
        → New task created
          → (EC2 launch type only) Capacity Provider adds EC2 if needed
```

## Exam Tips
- ECS scales on 3 metrics: **CPU**, **Memory**, **ALB Request Count Per Target**
- Uses **AWS Application Auto Scaling** (not EC2 Auto Scaling)
- **Fargate** = only task scaling needed (no EC2 to worry about) — exam prefers this
- EC2 launch type: prefer **ECS Cluster Capacity Provider** over ASG scaling
- Task scaling ≠ EC2 instance scaling — they are separate
