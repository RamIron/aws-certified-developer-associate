# 16.6 - Amazon ECS - Rolling Updates

## Key Concepts

### 1. What Are Rolling Updates?

A **deployment strategy** — when you deploy a **new version** of your app (e.g., update task definition from v1 to v2), ECS gradually replaces old tasks with new ones.

- This is NOT auto scaling (scaling = more/fewer tasks based on traffic)
- This is a **version replacement** (v1 → v2, same task count at the end)

| | Auto Scaling | Rolling Updates |
|---|-------------|-----------------|
| **When** | Traffic changes | You deploy a new version |
| **What changes** | Number of tasks (more/fewer) | Version of tasks (v1 → v2) |
| **End result** | Different task count | Same task count, new version |

### 2. Two Settings for Rolling Updates

| Setting | Default | What it means |
|---------|---------|--------------|
| **Minimum Healthy Percent** | 100% | How LOW can running capacity go during update |
| **Maximum Percent** | 200% | How HIGH can total capacity go during update |

### 2. Example: Min 50%, Max 100% (terminate first)

```
Start: [v1] [v1] [v1] [v1]  → 100% (4 tasks)
Step 1: [v1] [v1]            → 50%  (terminate 2 old)
Step 2: [v1] [v1] [v2] [v2] → 100% (create 2 new)
Step 3: [v2] [v2]            → 50%  (terminate 2 old)
Step 4: [v2] [v2] [v2] [v2] → 100% (create 2 new) ✅ done
```

- Can go **below** 100% → terminates old tasks first, then creates new ones

### 3. Example: Min 100%, Max 150% (create first)

```
Start: [v1] [v1] [v1] [v1]            → 100% (4 tasks)
Step 1: [v1] [v1] [v1] [v1] [v2] [v2] → 150% (create 2 new)
Step 2: [v1] [v1] [v2] [v2]           → 100% (terminate 2 old)
Step 3: [v1] [v1] [v2] [v2] [v2] [v2] → 150% (create 2 new)
Step 4: [v2] [v2] [v2] [v2]           → 100% (terminate 2 old) ✅ done
```

- Can **never** drop below 100% → creates new tasks first, then terminates old ones

---

## Exam Tips
- **Min 50%, Max 100%** = terminate old first, then create new (some downtime in capacity)
- **Min 100%, Max 150%** = create new first, then terminate old (no capacity loss, but uses more resources temporarily)
- Min 100% = "never go below current capacity" — safer for production
