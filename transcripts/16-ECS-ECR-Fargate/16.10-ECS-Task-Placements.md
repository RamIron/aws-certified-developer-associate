# 16.10 - Amazon ECS - Task Placements

## Key Concepts

### 1. Only for EC2 Launch Type

- Task placement = deciding **which EC2 instance** to put a task on (or remove from)
- **NOT for Fargate** — AWS handles placement for you
- It's a "best effort" process, not a guarantee

### 2. Task Placement Process

```
1. Filter: which instances meet CPU, memory, port requirements?
2. Apply: task placement constraints
3. Apply: task placement strategy
4. Place the task on the selected instance
```

### 3. Task Placement Strategies

| Strategy | What it does | Use case |
|----------|-------------|----------|
| **Binpack** | Fill one instance as much as possible before using the next | **Cost saving** (minimize number of instances) |
| **Random** | Place tasks randomly, no logic | Simple, no optimization |
| **Spread** | Spread tasks evenly across a value (AZ, instance ID) | **High availability** |

**Binpack** (cost saving — pack containers tightly):
```
Instance 1: [Task][Task][Task][Task]  ← fill this first
Instance 2: [Task]                    ← only when Instance 1 is full
Instance 3: (empty)
```

**Random:**
```
Instance 1: [Task]    [Task]
Instance 2:       [Task]
Instance 3: [Task]         [Task]    ← no pattern
```

**Spread on AZ** (high availability):
```
AZ-A:  [Task 1] [Task 4]
AZ-B:  [Task 2] [Task 5]
AZ-C:  [Task 3] [Task 6]    ← evenly distributed
```

- Strategies can be **combined**: e.g., spread on AZ + binpack on memory

### 4. Task Placement Constraints

| Constraint | What it does |
|-----------|-------------|
| **distinctInstance** | Each task on a **different** EC2 instance (never two tasks on same instance) |
| **memberOf** | Only place on instances matching an expression (e.g., only t2 instance types) |

**distinctInstance JSON:**
```json
"placementConstraints": [
  {
    "type": "distinctInstance"
  }
]
```

**memberOf JSON** (uses Cluster Query Language):
```json
"placementConstraints": [
  {
    "expression": "attribute:ecs.instance-type =~ t2.*",
    "type": "memberOf"
  }
]
```

---

## Exam Tips
- Task placement = **EC2 launch type only** (not Fargate)
- **Binpack** = cost saving (fill instances, minimize count)
- **Spread** = high availability (distribute across AZs)
- **Random** = no optimization
- **distinctInstance** = one task per instance
- **memberOf** = filter by instance attributes (e.g., instance type)
- Strategies can be mixed (spread on AZ + binpack on memory)
