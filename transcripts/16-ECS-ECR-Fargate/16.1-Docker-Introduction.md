# 16.1 - Docker Introduction

## Key Concepts

### 1. What is Docker?

- Software platform to deploy apps in **containers**
- Containers are standardized → run the same way on **any machine/OS**
- No compatibility issues, predictable behavior, easy to deploy
- Use cases: **microservices**, lift-and-shift to cloud, running any containerized app

### 2. How Docker Works

```
Server (e.g., EC2 Instance)
  └── Docker Agent (Daemon)
        ├── Container 1: Java App
        ├── Container 2: Node.js App
        ├── Container 3: Java App (another instance)
        └── Container 4: MySQL Database
```

- Multiple containers run on one server
- Can run multiple instances of the same app
- Containers share the host OS resources (lightweight)

### 3. Docker vs Virtual Machines

| | Docker Container | Virtual Machine (EC2) |
|---|-----------------|----------------------|
| **Architecture** | Host OS → Docker Daemon → Containers | Host OS → Hypervisor → Guest OS → Apps |
| **Weight** | Lightweight (shares host OS) | Heavier (each has its own guest OS) |
| **Isolation** | Shared networking/data possible | Fully isolated |
| **Density** | Many containers per server | Fewer VMs per server |

### 4. Docker Workflow

```
1. Write a Dockerfile (defines the container)
2. Build → creates a Docker Image
3. Push → store image in a repository (Docker Hub or Amazon ECR)
4. Pull → download image from repository
5. Run → image becomes a running Docker Container
```

### 5. Docker Image Repositories

| Repository | Type | Description |
|-----------|------|-------------|
| **Docker Hub** | Public | Base images (Ubuntu, MySQL, etc.) |
| **Amazon ECR** | Private (+ public option) | AWS-managed container registry |
| **ECR Public Gallery** | Public | Public images on AWS |

### 6. Docker Container Management on AWS

| Service | Description |
|---------|-------------|
| **Amazon ECS** | Elastic Container Service — AWS's own container platform |
| **Amazon EKS** | Elastic Kubernetes Service — managed Kubernetes (open source) |
| **AWS Fargate** | Serverless containers — works with ECS and EKS |
| **Amazon ECR** | Elastic Container Registry — stores Docker images |

---

## Exam Tips
- **Docker** = containers = standardized, portable, lightweight
- "Microservices architecture" → think Docker/ECS
- Docker images stored in **Docker Hub** (public) or **Amazon ECR** (AWS)
- **ECS** = AWS container platform, **EKS** = managed Kubernetes
- **Fargate** = serverless containers (works with both ECS and EKS)
- **ECR** = where you store your Docker images on AWS
