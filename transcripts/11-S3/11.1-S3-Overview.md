# 11.1 - S3 Overview

## Key Concepts

### 1. What is Amazon S3?

- **Infinitely scaling storage** — one of the main building blocks of AWS
- Many websites and AWS services use S3 as a backbone
- Use cases: backup, disaster recovery, archival, hybrid cloud storage, media hosting, data lakes, static websites, software updates

### 2. Buckets

- S3 stores files into **buckets** (top-level directories)
- Bucket names must be **globally unique** (across ALL regions and ALL accounts)
- Buckets are defined at the **region level** (S3 looks global in the console, but buckets live in a specific region)

**Naming rules:**
- No uppercase, no underscores
- 3-63 characters long
- Must not be an IP
- Must start with lowercase letter or number

### 3. Objects

- Objects = **files** stored in buckets
- Object **key** = the full path (e.g., `my-folder/another-folder/my-file.txt`)
- Key = **prefix** + **object name**
  - Prefix: `my-folder/another-folder/`
  - Object name: `my-file.txt`
- S3 has **no real directory concept** — it's just keys with slashes (the console simulates folders)

### 4. Object Properties

| Property | Details |
|----------|---------|
| **Max size** | **5 TB** (5,000 GB) |
| **Multi-part upload** | Required for files **> 5 GB** |
| **Metadata** | Key-value pairs (system or user-defined) |
| **Tags** | Up to **10** Unicode key-value pairs (useful for security & lifecycle) |
| **Version ID** | Only if versioning is enabled |

---

## Exam Tips
- Bucket names are **globally unique** across all AWS accounts
- Buckets are **region-level** resources (not global, despite the console UI)
- Object key = full path, S3 has no real directories
- Max object size = **5 TB**, multi-part upload required for **> 5 GB**
- Tags (up to 10) are useful for security and lifecycle policies
