# 11.12 - S3 Storage Classes Overview

## Key Concepts

### 1. Durability vs Availability

| Concept | What it means |
|---------|--------------|
| **Durability** | How likely you are to lose an object. **11 nines** (99.999999999%) for ALL classes — lose 1 object per 10 million every 10,000 years |
| **Availability** | How readily the service is accessible. **Varies** by storage class |

### 2. Storage Classes

| Class | Availability | Use Case | Retrieval |
|-------|-------------|----------|-----------|
| **S3 Standard** | 99.99% | Frequently accessed data (default) | Instant |
| **S3 Standard-IA** (Infrequent Access) | 99.9% | Infrequent access, rapid retrieval needed | Instant (retrieval cost) |
| **S3 One Zone-IA** (One Zone - Infrequent Access) | 99.5% | Secondary backups, recreatable data | Instant (retrieval cost) |
| **Glacier Instant Retrieval** | — | Data accessed ~once per quarter | **Milliseconds** (min 90 days) |
| **Glacier Flexible Retrieval** | — | Archives, willing to wait | **1-5 min / 3-5 hr / 5-12 hr** (min 90 days) |
| **Glacier Deep Archive** | — | Long-term archive, lowest cost | **12 hr / 48 hr** (min 180 days) |
| **S3 Intelligent-Tiering** | — | Unknown/changing access patterns | Auto-moves between tiers |

### 3. Glacier Retrieval Tiers

**Glacier Flexible Retrieval:**

| Tier | Time | Cost |
|------|------|------|
| **Expedited** | 1-5 minutes | Highest |
| **Standard** | 3-5 hours | Medium |
| **Bulk** | 5-12 hours | Free |

**Glacier Deep Archive** (only 2 tiers — NO Expedited!):

| Tier | Time |
|------|------|
| **Standard** | 12 hours |
| **Bulk** | 48 hours |

### 4. S3 Intelligent-Tiering

- Automatically moves objects between tiers based on usage
- Small monthly monitoring + auto-tiering fee
- **No retrieval charges**

| Tier | Trigger | Type |
|------|---------|------|
| Frequent Access | Default | Automatic |
| Infrequent Access | Not accessed 30 days | Automatic |
| Archive Instant Access | Not accessed 90 days | Automatic |
| Archive Access | 90-700+ days | Optional (configurable) |
| Deep Archive Access | 180-700+ days | Optional (configurable) |

### 5. Moving Between Classes

- Choose class when creating an object
- Modify manually at any time
- Use **S3 Lifecycle configurations** to move objects automatically
  - Example: Standard → Standard-IA after 30 days → Glacier after 180 days
- **Reduced Redundancy** storage class exists but is **deprecated** (ignore it)

---

## Exam Tips
- Durability is **11 nines for ALL classes** (same everywhere)
- Availability varies: Standard (99.99%) > Standard-IA (99.9%) > One Zone-IA (99.5%)
- **One Zone-IA** = single AZ, data lost if AZ destroyed — only for recreatable data
- Glacier Instant = milliseconds | Flexible = minutes to hours | Deep Archive = hours to days
- Glacier minimum storage: Instant & Flexible = **90 days**, Deep Archive = **180 days**
- Intelligent-Tiering = auto-moves, no retrieval charges, small monitoring fee
- "Rarely accessed but need instantly" → **Glacier Instant Retrieval**
- "Archive, can wait hours" → **Glacier Flexible**
- "Long-term, cheapest, can wait days" → **Glacier Deep Archive**
