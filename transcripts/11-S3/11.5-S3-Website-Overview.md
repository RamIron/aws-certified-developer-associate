# 11.5 - S3 Website Overview

## Key Concepts

### 1. S3 Static Website Hosting

- S3 can host **static websites** (HTML, CSS, JS, images) directly — no server needed
- You enable "Static Website Hosting" in the bucket properties
- Must specify an **index document** (e.g., `index.html`) — the homepage
- S3 gives you a website URL:
  - `http://bucket-name.s3-website-aws-region.amazonaws.com` (dash)
  - `http://bucket-name.s3-website.aws-region.amazonaws.com` (dot)

### 2. Requirements

- The bucket must have **public reads enabled** (bucket policy with `Principal: "*"`)
- Block Public Access must be **disabled**
- If you get **403 Forbidden** → bucket policy doesn't allow public reads

### 3. What "Static" Means

- Content is served as-is — no server-side processing
- No PHP, no database queries, no dynamic backend
- Use cases: landing pages, documentation sites, single-page apps (React/Angular builds)

---

## Exam Tips
- S3 can host static websites without EC2
- 403 Forbidden on S3 website = **bucket policy not public**
- S3 Website is a valid **Route 53 Alias target**
- "Static" = no server-side logic, just files
