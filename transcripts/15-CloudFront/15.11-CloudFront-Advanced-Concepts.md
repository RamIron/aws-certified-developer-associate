# 15.11 - CloudFront Advanced Concepts

## Key Concepts

### 1. Price Classes

Cost varies by edge location region (e.g., US is cheaper than India). You can reduce cost by limiting which edge locations are used:

| Price Class | Regions | Cost |
|-------------|---------|------|
| **All** | All edge locations worldwide | Highest (best performance) |
| **200** | Most regions, excludes the most expensive | Medium |
| **100** | North America + Europe only | Lowest |

```
Price Class 100: North America + Europe
Price Class 200: + South America, Africa, Asia (most)
Price Class All: Entire world
```

### 2. Origin Groups (Failover)

Increase high availability with a **primary** and **secondary** origin:

```
CloudFront → Primary Origin (EC2/S3) → Error?
                                        ├── NO  → serve response ✅
                                        └── YES → retry on Secondary Origin ✅
```

- One primary + one secondary per origin group
- If primary fails → automatic failover to secondary

### 3. Origin Groups with S3 (Regional Disaster Recovery)

```
Origin Group
  ├── Primary: S3 Bucket (us-east-1)
  └── Secondary: S3 Bucket (eu-west-1)
       ↑
  S3 Cross-Region Replication keeps them in sync
```

- If the primary region goes down → CloudFront fails over to the secondary bucket
- CRR ensures both buckets have the same data
- Achieves **region-level high availability**

### 4. Field-Level Encryption

Encrypt specific sensitive fields (up to **10 fields**) at the edge location, decryptable only by the final web server.

```
Client → Edge Location (encrypts credit card with public key)
           → CloudFront → ALB → Web Server
                                  ↑
                          decrypts with private key
```

- Uses **asymmetric encryption** (public key encrypts, private key decrypts)
- Everything travels over HTTPS, but the encrypted field is **additionally** protected
- ALB and CloudFront **cannot** read the encrypted field — only the web server with the private key
- Use case: credit card numbers, PII that must stay encrypted through the entire stack

---

## Exam Tips
- **Price Class 100** = cheapest (NA + Europe), **All** = best performance (global)
- **Origin Groups** = failover (primary + secondary origin)
- Origin Groups + S3 CRR = **regional disaster recovery**
- **Field-Level Encryption** = encrypt specific fields at the edge, only web server can decrypt
- "Protect sensitive data through entire application stack" → Field-Level Encryption
- "CloudFront high availability if origin fails" → Origin Groups
