# 15.5 - CloudFront - Cache Behaviors

## Key Concepts

### 1. What Are Cache Behaviors?

Route requests to **different origins** based on URL **path patterns**, each with its own cache settings.

```
CloudFront Distribution
    ├── /images/*  → S3 Bucket (origin 1)
    ├── /api/*     → ALB (origin 2)
    └── /*         → Default origin (always last)
```

- Each path pattern can have its own **origin**, **cache policy**, and **origin request policy**
- The **default cache behavior** (`/*`) is always processed **last** (fallback)
- More specific patterns are matched first

### 2. Use Case: Gating Access with Signed Cookies

```
1. User hits /login → Cache behavior routes to EC2 instance
2. EC2 generates CloudFront signed cookies → sent to user
3. User accesses /* (any other URL) with signed cookies → access granted → S3 files served
4. User tries /* WITHOUT signed cookies → redirected back to /login
```

- `/login` behavior → EC2 origin (generates signed cookies)
- `/*` default behavior → S3 origin (requires signed cookies)

### 3. Use Case: Maximize Cache Hits

| Path | Origin | Cache Strategy |
|------|--------|---------------|
| `/static/*` | S3 Bucket | No headers/cookies in Cache Key → maximum hits |
| `/api/*` | ALB + EC2 | Cache based on specific headers/cookies → more granular |

- Static content: simple Cache Key = high hit ratio
- Dynamic content: enhanced Cache Key with relevant headers/cookies

---

## Exam Tips
- Cache behaviors = route different URL patterns to different origins
- Default cache behavior (`/*`) is always **last** to be processed
- "Require authentication before accessing S3 content" → cache behavior with signed cookies
- "Static and dynamic content from same distribution" → separate cache behaviors with different origins and cache policies
