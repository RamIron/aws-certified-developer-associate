# 15.1 - CloudFront Overview

## Key Concepts

### 1. What is CloudFront?

- **Content Delivery Network (CDN)** — CDN in the exam = CloudFront
- Caches content at **edge locations** worldwide (~216 points of presence)
- Improves **read performance** → lower latency for users globally
- Provides **DDoS protection** (distributed content + AWS Shield + WAF)

### 2. CloudFront Origins (Backends)

| Origin Type | Description | Access |
|-------------|-------------|--------|
| **S3 Bucket** | Distribute & cache files at the edge, or upload files to S3 via CloudFront | Secured via **OAC** (Origin Access Control) + S3 bucket policy |
| **VPC Origin** | Private ALB, NLB, or EC2 instances in private subnets | Connects to resources inside your VPC |
| **Custom HTTP** | Any public HTTP endpoint (S3 static website, public ALB, etc.) | Direct HTTP connection |

### 3. How CloudFront Works

```
Client → Edge Location → Cache hit? → YES → serve from cache
                                    → NO  → fetch from Origin → cache result → serve
```

- Edge location checks its **local cache** first
- On cache miss: fetches from origin via **AWS private network**, then caches the result
- Next request for same content at same edge → served from cache (no origin trip)

### 4. S3 as Origin — Security

```
Users → Edge Location → (private network) → S3 Bucket
                                              ↑
                                    OAC + S3 Bucket Policy
```

- **Origin Access Control (OAC)** secures the connection between CloudFront and S3
- S3 bucket policy is modified to only allow access from CloudFront

### 5. CloudFront vs S3 Cross-Region Replication

| | CloudFront | S3 Cross-Region Replication |
|---|-----------|---------------------------|
| **Scope** | ~216 edge locations globally | Must set up per region |
| **Caching** | Files cached (e.g., ~1 day TTL) | Files updated in near real-time |
| **Best for** | Static content available **everywhere** | Dynamic content at low latency in **a few regions** |
| **Type** | CDN (cache) | Full bucket replication (read-only) |

---

## Exam Tips
- **CDN** in the exam = **CloudFront**
- Origins: S3 bucket (OAC), VPC origin (private resources), Custom HTTP
- S3 origin secured via **OAC** (Origin Access Control) + S3 bucket policy
- "Static content worldwide" → CloudFront
- "Dynamic content in a few regions" → S3 Cross-Region Replication
- CloudFront also provides **DDoS protection** via Shield + WAF
